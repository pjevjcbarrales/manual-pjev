Estándar de Reportes Complejos (JasperReports)

Manejo de documentos estructurados, con diseño pixel-perfect, agrupaciones complejas y generación en el lado del servidor.

1 ¿Cuándo usar JasperReports?

Se utiliza JasperReports exclusivamente cuando el reporte requiere:

  • Diseños Pixel-Perfect: Posicionamiento exacto de membretes, logos y firmas.
  • Documentos Contables y Oficiales: Pólizas contables, Afectaciones de gasto, Órdenes de pago.
  • Agrupaciones complejas: Sumatorias condicionales por página, cortes de control y grupos anidados con subtotales (Bands).
  • Generación de reportes voluminosos que podrían saturar la memoria del Frontend si se generaran en JS.
2 Flujo de Generación (Servidor)

Los reportes complejos se generan en el backend. El flujo es el siguiente:

[Frontend Next.js]
       │
       │ 1. Pide impresión (ej. /api/conta/poliza/123/pdf)
       ▼
[Backend Nest.js]
       │
       │ 2. Ejecuta Query SQL en PostgreSQL
       │ 3. Recupera datos de Póliza y Partidas
       │ 4. Invoca el compilador JasperReports (java -jar / jasperserver)
       │    con la plantilla .jrxml y los datos.
       ▼
[Motor JasperReports] -> 5. Retorna un Buffer / Blob PDF
       │
       ▼
[Frontend Next.js] -> 6. Descarga o Muestra en iframe visor
3 Reglas de Desarrollo (.jrxml)

Los desarrolladores encargados de los reportes complejos deben observar las siguientes reglas al utilizar Jaspersoft Studio:

!
Requisitos de Instalación: El entorno de backend requiere la instalación de la máquina virtual de Java (JVM) y las dependencias de JasperReports para poder compilar y exportar los archivos `.jrxml`.
  • Consultas SQL: La lógica pesada y los cruces (JOINs) deben hacerse en el Query principal dentro del reporte, no en fórmulas de Jasper.
  • Imágenes: Los logos y sellos deben pasarse como parámetros (Base64 o URL) para que la plantilla sea agnóstica de su ruta física.
  • Estructura: Siempre dividir el diseño en Title, PageHeader, ColumnHeader, Detail, PageFooter y Summary de manera limpia.
