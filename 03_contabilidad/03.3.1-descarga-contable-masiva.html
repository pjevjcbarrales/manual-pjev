03.3.1

Descarga Contable Masiva

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV. Procedimiento Crítico de Cierre.

Tipo: Proceso Batch (Cierre Mensual)
1 Descripción General

Es una de las herramientas más potentes del SIAF-PJEV. Permite al equipo de Contabilidad amortizar o "descargar" contablemente cientos de comprobaciones de viáticos, compras de almacén, depreciaciones de activo fijo u obras públicas de un solo clic al cierre del mes. Adicionalmente, el sistema le permite al usuario elegir el tipo de póliza para consultar por lotes lo que está pendiente de aplicar. En lugar de capturar pólizas manuales una por una, el sistema agrupa saldos de cuentas transitorias y genera pólizas masivas automatizadas que matan los saldos circulantes mandándolos a la cuenta de Gasto definitivo.

4 Modelo de Datos

Al tratarse de un proceso masivo, no se requiere la creación de un nuevo esquema de base de datos, sino que se ocupan los modelos de datos ya existentes correspondientes a contabilidad.poliza y contabilidad.poliza_movtos.

Para obtener la información contable válida para la descarga, el sistema aplica las siguientes reglas de selección:

  • Selecciona las pólizas que estén debidamente cuadradas.
  • Que no estén afectadas.
  • Que se encuentren autorizadas (estatus = 3).
  • Que no estén canceladas.
  • Que estén dentro del periodo activo (mes + ejercicio).

Definición SQL (Snippet Filtros Fijos)

-- ==========================================
-- 1. LEER PÓLIZAS PARA DESCARGA
-- ==========================================
WHERE p.cuadrada = 1 
  AND p.afectada = 0 
  AND p.estatus = 3 -- (Autorizada, No cancelada)
  AND p.fecha BETWEEN @p_fecha_inicio AND @p_fecha_fin; -- (Dentro del periodo activo)

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas:

  • Afectación Masiva: Dispara el Job de afectación en bloque de las pólizas seleccionadas. Al ser intensivo, muestra una barra de carga en la interfaz.
  • Desafectación Masiva: Revierte o cancela el impacto de un bloque de pólizas seleccionadas previamente afectadas.
  • Impresión / Exportación: Botón 🖨️ que genera el "Reporte Analítico", desglosando los registros procesados en esa corrida específica (Crucial para auditoría y anexarlo a la póliza).

10 Flujos de Integración

Es un proceso dependiente. Requiere que otros módulos (Viáticos, Recursos Materiales, Activo Fijo) hayan finalizado sus capturas y comprobaciones de fin de mes. Al ejecutarse este proceso batch, inyecta una póliza gigante al módulo 03.2.1 Visor de Pólizas Automáticas, el cual a su vez alimentará la Balanza de Comprobación depurando los saldos transitorios.

PROCESO / ÁREA JEFE DE CONTABILIDAD SIAF-PJEV
1. SIMULACIÓN PREVIA Ingresa los parámetros (tipo y fecha) y pide la simulación de lo que amortizará el sistema. Ejecuta SP de solo lectura que barre las tablas de viáticos y almacén. Muestra los totales propuestos.
2. VERIFICACIÓN Analiza si el importe es coherente con sus expectativas de fin de mes. Si está correcto, ordena la ejecución.
3. EJECUCIÓN DEL JOB BATCH Espera a que el sistema procese (ve la barra de progreso). Crea N renglones en una póliza consolidada y marca las filas de origen como `descargado=1` para no volverlas a procesar.
4. GENERACIÓN DE REPORTES Presiona Imprimir para descargar el anexo que soportará la macro-póliza. Exporta el PDF de cientos de páginas con el nivel de detalle requerido por el ORFIS.
