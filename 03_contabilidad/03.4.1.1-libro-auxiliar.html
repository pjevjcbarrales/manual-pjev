03.4.1.1

Libro Auxiliar Contable

Reporte oficial exigido por el CONAC (Normativa SIAF) que detalla analíticamente los movimientos de cada cuenta contable en un periodo determinado.

Tipo: Reporte Financiero (DOC-08)
1 Descripción General

El Libro Auxiliar es el nivel de mayor granularidad de la contabilidad. Muestra el saldo inicial de una o varias cuentas, seguido por todos los cargos y abonos que afectaron a la cuenta, detallando la fecha, número de póliza, y concepto, hasta llegar al saldo final del periodo. Su estructura es indispensable para las labores de auditoría (ORFIS) y conciliación.

2 Marco Normativo (SIAF / CONAC)
NormaObligación que genera
Ley General de Contabilidad Gubernamental (LGCG)Obliga a los entes a mantener libros contables, siendo el Auxiliar el respaldo analítico del Libro Mayor y del Diario.
Manual de Contabilidad Gubernamental (CONAC)Establece que el Libro Auxiliar debe contener: Fecha, Número y Tipo de Póliza, Concepto, Saldo Inicial, Cargos, Abonos y Saldo Final por cada cuenta o subcuenta de nivel 5 o desagregación posterior.
3 Roles que intervienen
RolAcción permitidaRestricción
Analista Contable / Jefe de ContabilidadPuede generar el libro auxiliar definiendo rangos de cuentas, meses o fechas específicas y centros de costo.No puede alterar la información. Reporte de solo lectura.
Auditor Externo (ORFIS)Extraer la sábana analítica en Excel para pruebas de cumplimiento.Acceso de lectura en módulo auditor.
4 Modelo de Datos

El libro auxiliar consolida los saldos iniciales y los movimientos agrupados por niveles de afectación. Se definen dos tablas persistentes en el esquema contabilidad que usan la columna id_sesion para asegurar la concurrencia web (cada usuario inserta/borra sus propios registros de reporte).

Definición Funcional (PostgreSQL PL/pgSQL Completo)

-- ==========================================
-- Genera Libro Auxiliar Contable (Migración a PG)
-- Tablas: contabilidad.rep_auxiliar_encabezado y detalle
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_genera_auxiliar_usuario(
    p_partida_inicio VARCHAR,
    p_partida_final VARCHAR,
    p_inicio_ano VARCHAR,
    p_fecha_inicio VARCHAR,
    p_fecha_final VARCHAR,
    p_empleado VARCHAR,
    p_proveedor VARCHAR,
    p_obra VARCHAR,
    p_ua_inicial VARCHAR,
    p_ua_final VARCHAR,
    p_conciliado SMALLINT,
    p_usuario VARCHAR,
    p_sesion INT
) RETURNS VOID AS $$
DECLARE
    v_num_poliza_inicial INT;
BEGIN
    -- 1. Obtengo el número de la póliza de apertura
    SELECT COALESCE(numero_poliza, 0) INTO v_num_poliza_inicial
    FROM contabilidad.polizas_contables
    WHERE folio = '1' AND tipo = 'PD';

    IF NOT FOUND THEN
        v_num_poliza_inicial := 0;
    END IF;

    -- 2. Limpieza de tablas de reporte para la sesión actual (Sustituye SQL Dinámico de SQL Server)
    DELETE FROM contabilidad.rep_auxiliar_encabezado WHERE id_sesion = p_sesion;
    DELETE FROM contabilidad.rep_auxiliar_detalle WHERE id_sesion = p_sesion;

    -- 3. Inserción Nivel 0 (Cuentas sin afectación a terceros)
    INSERT INTO contabilidad.rep_auxiliar_encabezado(
        numero_poliza, partida, saldo_inicial, folio, tipo, descripcion, id_sesion
    )
    SELECT
        COALESCE((SELECT MAX(numero_poliza) FROM contabilidad.rep_auxiliar_encabezado WHERE id_sesion = p_sesion), 0) + ROW_NUMBER() OVER(ORDER BY cs.cadena_completa) + 1,
        cs.cadena_completa,
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pt.acumulada = 1 AND pt.folio = '1' AND pt.tipo = 'PD'
        ) +
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pt.acumulada = 1 AND NOT (pt.folio = '1' AND pt.tipo = 'PD')
            AND pt.fecha::date < p_fecha_inicio::date
        ),
        cs.cadena_completa,
        'PE',
        cs.descripcion,
        p_sesion
    FROM contabilidad.cat_subsubsubpartidas cs
    WHERE cs.cadena_completa BETWEEN p_partida_inicio AND p_partida_final
      AND EXISTS (SELECT 1 FROM contabilidad.polizas_contables_detalle pd WHERE pd.cuenta_completa = cs.cadena_completa)
      AND cs.nivel_afectacion = 0;

    -- 4. Inserción Nivel 1 (Empleado)
    INSERT INTO contabilidad.rep_auxiliar_encabezado(
        numero_poliza, partida, saldo_inicial, folio, tipo, descripcion, id_sesion
    )
    SELECT
        COALESCE((SELECT MAX(numero_poliza) FROM contabilidad.rep_auxiliar_encabezado WHERE id_sesion = p_sesion), 0) + ROW_NUMBER() OVER(ORDER BY cs.cadena_completa) + 1,
        cs.cadena_completa,
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.empleado = pd.empleado AND pt.acumulada = 1 AND pt.folio = '1' AND pt.tipo = 'PD'
        ) +
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.empleado = pd.empleado AND pt.acumulada = 1 AND NOT (pt.folio = '1' AND pt.tipo = 'PD')
            AND pt.fecha::date < p_fecha_inicio::date
        ),
        pd.empleado::varchar,
        'PE',
        (SELECT emp.numero_empleado::varchar || ' ' || emp.nombre_completo FROM contabilidad.cat_empleados emp WHERE emp.numero_empleado = pd.empleado LIMIT 1),
        p_sesion
    FROM contabilidad.cat_subsubsubpartidas cs
    JOIN contabilidad.polizas_contables_detalle pd ON cs.cadena_completa = pd.cuenta_completa
    JOIN contabilidad.polizas_contables p ON p.numero_poliza = pd.numero_poliza
    WHERE cs.cadena_completa BETWEEN p_partida_inicio AND p_partida_final
      AND cs.nivel_afectacion = 1
      AND p.fecha::date BETWEEN TO_DATE(p_inicio_ano, 'YYYY-MM-DD') AND p_fecha_final::date
      AND (pd.empleado = p_empleado::int OR p_empleado = '0')
    GROUP BY cs.cadena_completa, cs.naturaleza, cs.descripcion, cs.nivel_afectacion, pd.empleado;

    -- 5. Inserción Nivel 2 (Proveedor)
    INSERT INTO contabilidad.rep_auxiliar_encabezado(
        numero_poliza, partida, saldo_inicial, folio, tipo, descripcion, id_sesion
    )
    SELECT
        COALESCE((SELECT MAX(numero_poliza) FROM contabilidad.rep_auxiliar_encabezado WHERE id_sesion = p_sesion), 0) + ROW_NUMBER() OVER(ORDER BY cs.cadena_completa),
        cs.cadena_completa,
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.proveedor = pd.proveedor AND pt.acumulada = 1 AND pt.folio = '1' AND pt.tipo = 'PD'
        ) +
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.proveedor = pd.proveedor AND pt.acumulada = 1 AND NOT (pt.folio = '1' AND pt.tipo = 'PD')
            AND pt.fecha::date < p_fecha_inicio::date
        ),
        pd.proveedor::varchar,
        'PE',
        (SELECT prov.numero_proveedor::varchar || ' ' || prov.razon_social FROM contabilidad.cat_proveedores prov WHERE prov.numero_proveedor = pd.proveedor LIMIT 1),
        p_sesion
    FROM contabilidad.cat_subsubsubpartidas cs
    JOIN contabilidad.polizas_contables_detalle pd ON cs.cadena_completa = pd.cuenta_completa
    JOIN contabilidad.polizas_contables p ON p.numero_poliza = pd.numero_poliza
    WHERE cs.cadena_completa BETWEEN p_partida_inicio AND p_partida_final
      AND cs.nivel_afectacion = 2
      AND p.fecha::date BETWEEN TO_DATE(p_inicio_ano, 'YYYY-MM-DD') AND p_fecha_final::date
      AND (pd.proveedor = p_proveedor::int OR p_proveedor = '0')
    GROUP BY cs.cadena_completa, cs.naturaleza, cs.descripcion, cs.nivel_afectacion, pd.proveedor;

    -- 6. Inserción Nivel 3 (Obra / Municipio)
    INSERT INTO contabilidad.rep_auxiliar_encabezado(
        numero_poliza, partida, saldo_inicial, folio, tipo, descripcion, id_sesion
    )
    SELECT
        COALESCE((SELECT MAX(numero_poliza) FROM contabilidad.rep_auxiliar_encabezado WHERE id_sesion = p_sesion), 0) + ROW_NUMBER() OVER(ORDER BY cs.cadena_completa),
        cs.cadena_completa,
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.municipio = pd.municipio AND pt.acumulada = 1 AND pt.folio = '1' AND pt.tipo = 'PD'
        ) +
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.municipio = pd.municipio AND pt.acumulada = 1 AND NOT (pt.folio = '1' AND pt.tipo = 'PD')
            AND pt.fecha::date < p_fecha_inicio::date
        ),
        pd.municipio::varchar,
        'PE',
        'OBRA',
        p_sesion
    FROM contabilidad.cat_subsubsubpartidas cs
    JOIN contabilidad.polizas_contables_detalle pd ON cs.cadena_completa = pd.cuenta_completa
    JOIN contabilidad.polizas_contables p ON p.numero_poliza = pd.numero_poliza
    WHERE cs.cadena_completa BETWEEN p_partida_inicio AND p_partida_final
      AND cs.nivel_afectacion = 3
      AND p.fecha::date BETWEEN TO_DATE(p_inicio_ano, 'YYYY-MM-DD') AND p_fecha_final::date
      AND (pd.municipio = p_obra::int OR p_obra = '0')
    GROUP BY cs.cadena_completa, cs.naturaleza, cs.descripcion, cs.nivel_afectacion, pd.municipio;

    -- 7. Inserción Nivel 4 (Unidad Administrativa)
    INSERT INTO contabilidad.rep_auxiliar_encabezado(
        numero_poliza, partida, saldo_inicial, folio, tipo, descripcion, id_sesion
    )
    SELECT
        COALESCE((SELECT MAX(numero_poliza) FROM contabilidad.rep_auxiliar_encabezado WHERE id_sesion = p_sesion), 0) + ROW_NUMBER() OVER(ORDER BY cs.cadena_completa),
        cs.cadena_completa,
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.unidad_administrativa = pd.unidad_administrativa AND pt.acumulada = 1 AND pt.folio = '1' AND pt.tipo = 'PD'
        ) +
        (
            SELECT COALESCE(SUM(CASE WHEN cs.naturaleza = 'D' THEN pdt.debe - pdt.haber ELSE pdt.haber - pdt.debe END), 0)
            FROM contabilidad.polizas_contables pt
            JOIN contabilidad.polizas_contables_detalle pdt ON pt.numero_poliza = pdt.numero_poliza
            WHERE pdt.cuenta_completa = cs.cadena_completa AND pdt.unidad_administrativa = pd.unidad_administrativa AND pt.acumulada = 1 AND NOT (pt.folio = '1' AND pt.tipo = 'PD')
            AND pt.fecha::date < p_fecha_inicio::date
        ),
        pd.unidad_administrativa::varchar,
        'PE',
        (SELECT dep.clave_area || ' ' || dep.descripcion FROM contabilidad.cat_departamentos dep WHERE dep.numero_departamento = pd.unidad_administrativa LIMIT 1),
        p_sesion
    FROM contabilidad.cat_subsubsubpartidas cs
    JOIN contabilidad.polizas_contables_detalle pd ON cs.cadena_completa = pd.cuenta_completa
    JOIN contabilidad.polizas_contables p ON p.numero_poliza = pd.numero_poliza
    WHERE cs.cadena_completa BETWEEN p_partida_inicio AND p_partida_final
      AND cs.nivel_afectacion = 4
      AND p.fecha::date BETWEEN TO_DATE(p_inicio_ano, 'YYYY-MM-DD') AND p_fecha_final::date
      AND (pd.unidad_administrativa BETWEEN p_ua_inicial::int AND p_ua_final::int OR p_ua_inicial = '0' OR p_ua_final = '0')
    GROUP BY cs.cadena_completa, cs.naturaleza, cs.descripcion, cs.nivel_afectacion, pd.unidad_administrativa;

    -- 8. Inserción Detalle de Pólizas relacionadas
    INSERT INTO contabilidad.rep_auxiliar_detalle(
        debe, haber, unidad_administrativa, empleado, proveedor, municipio, 
        cuenta, scuenta, sscuenta, ssscuenta, sssscuenta, numero_poliza, 
        fecha_captura, poliza_original, id_sesion, cuenta_completa
    )
    SELECT
        pd.debe, pd.haber, pd.unidad_administrativa, pd.empleado, pd.proveedor, pd.municipio,
        pd.cuenta, pd.scuenta, pd.sscuenta, pd.ssscuenta, pd.sssscuenta,
        CASE 
            WHEN cs.nivel_afectacion = 0 THEN (SELECT pa.numero_poliza FROM contabilidad.rep_auxiliar_encabezado pa WHERE pa.partida = pd.cuenta_completa AND pa.folio = pd.cuenta_completa AND pa.id_sesion = p_sesion LIMIT 1)
            WHEN cs.nivel_afectacion = 1 THEN (SELECT pa.numero_poliza FROM contabilidad.rep_auxiliar_encabezado pa WHERE pa.partida = pd.cuenta_completa AND pa.folio = pd.empleado::varchar AND pa.id_sesion = p_sesion LIMIT 1)
            WHEN cs.nivel_afectacion = 2 THEN (SELECT pa.numero_poliza FROM contabilidad.rep_auxiliar_encabezado pa WHERE pa.partida = pd.cuenta_completa AND pa.folio = pd.proveedor::varchar AND pa.id_sesion = p_sesion LIMIT 1)
            WHEN cs.nivel_afectacion = 3 THEN (SELECT pa.numero_poliza FROM contabilidad.rep_auxiliar_encabezado pa WHERE pa.partida = pd.cuenta_completa AND pa.folio = pd.municipio::varchar AND pa.id_sesion = p_sesion LIMIT 1)
            WHEN cs.nivel_afectacion = 4 THEN (SELECT pa.numero_poliza FROM contabilidad.rep_auxiliar_encabezado pa WHERE pa.partida = pd.cuenta_completa AND pa.folio = pd.unidad_administrativa::varchar AND pa.id_sesion = p_sesion LIMIT 1)
        END AS numero_poliza,
        p.fecha_captura, p.numero_poliza AS poliza_original, p_sesion AS id_sesion,
        pd.cuenta_completa
    FROM contabilidad.polizas_contables p
    JOIN contabilidad.polizas_contables_detalle pd ON p.numero_poliza = pd.numero_poliza
    JOIN contabilidad.cat_subsubsubpartidas cs ON pd.cuenta_completa = cs.cadena_completa
    WHERE pd.cuenta_completa BETWEEN p_partida_inicio AND p_partida_final
      AND p.fecha::date BETWEEN p_fecha_inicio::date AND p_fecha_final::date
      AND p.acumulada <> 0
    ORDER BY p.numero_poliza;

END;
$$ LANGUAGE plpgsql;
6 Servicios REST

Debido a la arquitectura de concurrencia basada en id_sesion, el proceso de obtención del libro auxiliar se divide en dos pasos: Generación (POST) y Consulta/Exportación (GET).

MétodoEndpointBody / ParamsDescripción
POST/api/contabilidad/reportes/libro-auxiliar/generar{ "partida_inicio": "...", "partida_final": "...", "inicio_ano": "...", "fecha_inicio": "...", "fecha_final": "...", "empleado": "0", "proveedor": "0", "obra": "0", "ua_inicial": "0", "ua_final": "0", "conciliado": 1 }Ejecuta fn_genera_auxiliar_usuario. El p_sesion se inyecta en el backend a partir del token JWT del usuario. Devuelve un status OK indicando que las tablas temporales están listas.
GET/api/contabilidad/reportes/libro-auxiliar/consultar?token=...Consulta las tablas rep_auxiliar_encabezado y rep_auxiliar_detalle filtrando por el id_sesion del usuario para devolver el JSON anidado que pinta la pantalla.
GET/api/contabilidad/reportes/libro-auxiliar/exportar/pdf?token=...Genera el documento oficial PDF en Base64 o FileStream usando los datos pre-cargados en la sesión.
GET/api/contabilidad/reportes/libro-auxiliar/exportar/excel?token=...Descarga la sábana analítica en formato .xlsx usando los datos pre-cargados en la sesión.
7 Acciones del Módulo

El sistema expone las siguientes acciones operativas:

  • Filtros Complejos: El usuario puede cruzar filtros por: Rango de Cuentas, Rango de Fechas, o por Centro de Costo (Organización), permitiendo auxiliares por unidad administrativa.
  • Navegación Interactiva (Drill-down): Al presentar los movimientos en pantalla, dar clic en el número de póliza debe abrir una ventana modal o redireccionar al "Visor de Pólizas" para revisar el soporte documental.
  • Generación de PDF CONAC: Genera el documento con membrete, título "Libro Auxiliar", encabezados exactos estipulados por la norma, y área de firmas del Director de Contabilidad.

9 Reglas de Negocio Críticas
  • El reporte debe calcular dinámicamente el Saldo Inicial al corte del periodo solicitado.
  • Solo se consideran pólizas contables en estatus "Aprobada" o "Aplicada" (excluye borradores o canceladas).
  • La seguridad y concurrencia es manejada nativamente en PostgreSQL (excluyendo lecturas sucias).
  • Para Centros de Costo (Organización) no autorizados al usuario en sesión, el sistema omite dichos registros (seguridad por rol y fila).
10 Flujos de Integración (Impacto Transversal)

El Libro Auxiliar interactúa transversalmente con los siguientes módulos y catálogos maestros:

  • Nómina y Recursos Humanos: Catálogo de Empleados.
  • Proveedores (Padrón): Padrón de Proveedores y Contratistas.
  • Presupuesto: Catálogo de Unidades Administrativas, Centros de Costo y Estructura Programática.
11 Entregables / Reportes
  • Grid Interactivo (Visor Limpio): Despliega los movimientos en pantalla con opción Drill-down a la Póliza.
  • Formato PDF (CONAC): Exportación con estructura definida por el Consejo Nacional de Armonización Contable (encabezados formales y área de firmas).
  • Formato EXCEL: Exportación de datos planos (raw data) para análisis y conciliación manual.
