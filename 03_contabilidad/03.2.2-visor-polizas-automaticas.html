03.2.2

Visor de Pólizas de Diario Automatizadas

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV (Referencia: Manual PJEV 246-248).

Tipo: Formulario Compuesto (Lectura y Afectación)
1 Descripción General

Este módulo es el corazón de la integración del SIAF-PJEV. Permite al Departamento de Contabilidad revisar y afectar las pólizas que han sido generadas automáticamente por los módulos operativos (Presupuesto, Cuentas por Pagar, Tesorería/Caja, Recursos Humanos) a través de la Matriz de Conversión. El contador no digita cuentas ni importes, su rol es auditar y cruzar la información antes de consolidarla en la contabilidad general.

2 Marco Normativo
NormaArtículo / PuntoObligación que genera
LGCGPrincipio de Registro en Base AcumulativaLa contabilización de las transacciones de gasto e ingreso debe registrarse conforme a la fecha de su realización contable (devengo), independientemente de la de su pago o cobro.

Definición SQL (PostgreSQL)

-- ==========================================
-- 1. LEER ENCABEZADO
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_leer_encabezado(
    p_accion integer,
    p_ejercicio integer,
    p_usuario varchar,
    p_id integer,
    p_tipo varchar DEFAULT NULL,
    p_numero varchar DEFAULT NULL,
    p_fecha date DEFAULT NULL,
    p_descripcion varchar DEFAULT NULL,
    p_afecta integer DEFAULT NULL,
    p_centro_costo_id integer DEFAULT NULL,
    p_filtro varchar DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    IF p_accion = 0 THEN
        IF p_id < 0 THEN
            SELECT jsonb_build_object('p_id', 0, 'p_ejercicio', p_ejercicio, 'p_fecha', current_date, 'p_tipo', 'D', 'p_subtipo', '', 'p_numero', '', 'p_descripcion', '', 'p_centro_costo_id', NULL, 'p_afecta', 0, 'estatus_registro', 1) INTO v_resultado;
        ELSIF p_id > 0 THEN
            SELECT row_to_json(p) INTO v_resultado FROM contabilidad.poliza p WHERE p.p_id = p_id;
        ELSE
            SELECT jsonb_build_object('registros', COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb)) INTO v_resultado
              FROM (SELECT * FROM contabilidad.poliza WHERE estatus_registro = 1 AND p_ejercicio = p_ejercicio AND (p_filtro IS NULL OR p_descripcion ILIKE '%' || p_filtro || '%')) t;
        END IF;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'OK', 'data', v_resultado);
EXCEPTION WHEN OTHERS THEN
    RETURN jsonb_build_object('cod_resultado', -2, 'msg_resultado', SQLERRM, 'data', NULL);
END;
$$;

-- ==========================================
-- 2. ABC ENCABEZADO
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_abc_encabezado(
    p_accion integer, p_ejercicio integer, p_usuario varchar, p_id integer, 
    p_tipo varchar, p_numero varchar, p_fecha date, p_descripcion varchar, 
    p_afecta integer, p_centro_costo_id integer
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_id integer := p_id;
BEGIN
    IF p_accion = 1 THEN
        INSERT INTO contabilidad.poliza(p_ejercicio, p_tipo, p_numero, p_fecha, p_descripcion, p_centro_costo_id, p_afecta, estatus_registro, usuario_c)
        VALUES (p_ejercicio, p_tipo, p_numero, p_fecha, p_descripcion, p_centro_costo_id, p_afecta, 1, p_usuario) RETURNING p_id INTO v_id;
    ELSIF p_accion = 2 THEN
        UPDATE contabilidad.poliza SET p_fecha = p_fecha, p_descripcion = p_descripcion, p_centro_costo_id = p_centro_costo_id, p_afecta = p_afecta, usuario_m = p_usuario, fecha_m = now() WHERE p_id = p_id;
    ELSIF p_accion = 3 THEN
        UPDATE contabilidad.poliza SET estatus_registro = 0, usuario_m = p_usuario, fecha_m = now() WHERE p_id = p_id;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'Guardado', 'data', jsonb_build_object('id', v_id));
EXCEPTION WHEN OTHERS THEN
    RETURN jsonb_build_object('cod_resultado', -2, 'msg_resultado', SQLERRM);
END;
$$;

-- ==========================================
-- 3. LEER MOVTOS
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_leer_movtos(
    p_accion integer, p_ejercicio integer, p_usuario varchar, 
    p_id integer, p_poliza_id integer DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    IF p_accion = 0 THEN
        IF p_id < 0 THEN
            SELECT jsonb_build_object('pm_id', 0, 'pm_poliza_id', p_poliza_id, 'pm_debe', 0, 'pm_haber', 0, 'pm_cc_id', NULL, 'cc_descripcion', '') INTO v_resultado;
        ELSIF p_id > 0 THEN
            SELECT row_to_json(t) INTO v_resultado
              FROM (SELECT m.*, c.cc_descripcion, c.cc_cuentacompleta FROM contabilidad.poliza_movtos m LEFT JOIN contabilidad.catalogo_cuentas c ON m.pm_cc_id = c.cc_id WHERE m.pm_id = p_id) t;
        ELSE
            SELECT jsonb_build_object('registros', COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb)) INTO v_resultado
              FROM (SELECT m.*, c.cc_descripcion, c.cc_cuentacompleta FROM contabilidad.poliza_movtos m LEFT JOIN contabilidad.catalogo_cuentas c ON m.pm_cc_id = c.cc_id WHERE m.pm_poliza_id = p_poliza_id AND m.estatus_registro = 1 ORDER BY m.pm_orden) t;
        END IF;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

Definición SQL (PostgreSQL)

-- ==========================================
-- 4. ABC MOVTOS
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_abc_movtos(
    p_accion integer, p_ejercicio integer, p_usuario varchar, p_id integer, 
    p_poliza_id integer, p_cc_id integer, p_debe numeric, p_haber numeric
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_id integer := p_id;
BEGIN
    IF p_debe < 0 OR p_haber < 0 THEN RETURN jsonb_build_object('cod_resultado', -1, 'msg_resultado', 'Importe negativo.'); END IF;
    IF p_accion = 1 THEN
        INSERT INTO contabilidad.poliza_movtos(pm_ejercicio, pm_poliza_id, pm_cc_id, pm_debe, pm_haber, estatus_registro, usuario_c)
        VALUES (p_ejercicio, p_poliza_id, p_cc_id, p_debe, p_haber, 1, p_usuario) RETURNING pm_id INTO v_id;
    ELSIF p_accion = 2 THEN
        UPDATE contabilidad.poliza_movtos SET pm_cc_id = p_cc_id, pm_debe = p_debe, pm_haber = p_haber, usuario_m = p_usuario, fecha_m = now() WHERE pm_id = p_id;
    ELSIF p_accion = 3 THEN
        UPDATE contabilidad.poliza_movtos SET estatus_registro = 0, usuario_m = p_usuario, fecha_m = now() WHERE pm_id = p_id;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'Guardado', 'data', jsonb_build_object('id', v_id));
END;
$$;

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas (Maestro-Detalle):

  • pantalla_leer (Encabezados): Listado principal con grid de pólizas.
  • pantalla_abc (Encabezado): Formulario para capturar/editar Fecha, Concepto y Centro de Costos de la póliza.
  • pantalla_leer_detalle (Movimientos): Grid incrustado en la póliza que despliega todos los asientos registrados. Muestra la suma del Debe y Haber para validar cuadratura.
  • pantalla_abc_detalle (Movimientos): Modal o formulario anidado para agregar un Cargo/Abono, afectando a terceros si la cuenta lo exige.

10 Flujos de Integración

A diferencia de las automáticas, las pólizas manuales no tienen un documento "origen" en otros subsistemas. Su existencia nace y muere en la Subdirección de Contabilidad. Hacia adelante, se comportan exactamente igual: una vez en estatus "Afectada", sus importes se acumulan en la Balanza de Comprobación alimentando a los Estados Financieros de forma directa.

PROCESO / ÁREA ANALISTA CONTABLE SIAF-PJEV
1. IDENTIFICACIÓN NECESIDAD Detecta un error de registro anterior o necesidad de depreciación, junta los papeles de trabajo como evidencia.
2. CAPTURA Ingresa fecha, concepto, y agrega los renglones digitando cargo y abono. Suma matemáticamente los renglones en tiempo real. Habilita o deshabilita el botón Guardar si encuentra diferencia.
3. AFECTACIÓN Revisa que los datos estén bien capturados y presiona Afectar. Genera el folio automático M-XX, bloquea la póliza y suma los importes a los saldos del mayor (Balanza).
4. GENERACIÓN DE REPORTES Presiona Imprimir para recabar las firmas de "Elaboró, Revisó, Autorizó" y anexa sus papeles de trabajo físicos. Exporta el PDF formal de la póliza de diario con el logo institucional.

7 Stored Procedures Relacionados

Para mantener la modularidad, se utilizarán procedimientos específicos para leer desde los orígenes y ejecutar el copiado transaccional.

1. fn_leer_polizas_origen( @p_origen, @p_origen_id, @p_tipo, @p_numero, @p_fecha, ... )

Descripción: Consulta las pólizas generadas por el motor automático. Puede leer directamente de una vista consolidada (vw_polizas_precontables) que agrupa los módulos operativos, o leer de contabilidad.poliza si ya fueron afectadas.

Reglas que valida:

  • Si @p_origen = 99, consulta de todos los módulos disponibles. El @p_origen_id viaja como NULL en búsquedas generales.
  • Aplica paginación estándar.
  • Devuelve el campo fuente_sistema indicando de dónde proviene.

2. fn_afectar_poliza_automatizada( @p_origen, @p_origen_id, @p_ejercicio, @p_usuario )

Descripción: Ejecuta la regla crítica de segregación. Clona la póliza origen hacia el esquema contable y marca el origen como procesado.

Reglas que valida:

  • Busca la póliza en el esquema correspondiente usando p_modulo_origen y p_origen_id.
  • Valida que los renglones (movimientos origen) cuadren matemáticamente (Debe = Haber).
  • Realiza el INSERT INTO contabilidad.poliza y INSERT INTO contabilidad.poliza_movtos.
  • Realiza el UPDATE [esquema_origen].[tabla_polizas] SET poliza_contable_id = NUEVO_ID, estatus = 'Contabilizada'.
  • Suma los importes a la balanza de comprobación (Saldos del mayor).

3. fn_leer_movtos_origen( p_modulo_origen, p_origen_id )

Descripción: Consulta los renglones (Cuentas, Debe, Haber) que propuso la Matriz de Conversión para esa póliza específica.
