03.6.1

Cierre Mensual Contable

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV. Procedimiento de auditoría y cierre lógico del periodo contable.

Tipo: Panel de Control (Cierre Mensual)
1 Descripción General

El módulo "Cierre Mensual Contable" es el panel de control (Dashboard) que permite a la Dirección de Contabilidad auditar el estado de las pólizas contables generadas durante un mes y un ejercicio específico, y proceder con el cierre lógico del periodo. Cierra el mes una vez que no existan pólizas pendientes de afectar, impidiendo así cualquier modificación futura a la contabilidad de dicho periodo.

2 Marco Normativo (SIAF / CONAC)
NormaObligación que genera
Manual de Contabilidad Gubernamental (CONAC)El cierre contable mensual es obligatorio para garantizar la inmutabilidad de la información financiera una vez que se emiten los Estados Financieros. Un periodo cerrado no puede recibir nuevas pólizas ni modificaciones.
Reglas de Inmutabilidad de SIAFAsegura el cumplimiento de la regla de inmutabilidad: "Los registros contables no podrán ser alterados una vez que el ente público haya emitido la información financiera del periodo".

4 Modelo de Datos

Daryl, no debes crear tablas nuevas. Este módulo extrae la información de 3 fuentes exactas:

  • Para obtener el Ejercicio Activo: Usarás la tabla config.catalogo_ejercicios filtrando por eje_activo = true.
  • Para la Tabla de Control de Meses: Usarás la tabla config.catalogo_periodos. Un mes se considera CERRADO cuando per_estatus = true.
  • Para la Estadística de Pólizas: Generarás una vista dinámica leyendo polizamovto, filtrando por estatus < 4.

Vista Dinámica de Estadísticas

-- Consulta base para devolver la matriz de datos al frontend.
SELECT 
    e.eje_anio AS ejercicio_anio,
    p.per_ejercicio_id AS ejercicio, 
    p.per_mes AS mes, 
    p.per_estatus AS mes_cerrado,
    p.per_enviado_contabilidad AS mes_calculado,
    SUM(CASE WHEN m.afecta = 1 THEN m.poliza_pd ELSE 0 END) AS pd_afectadas,
    SUM(CASE WHEN m.afecta = 0 THEN m.poliza_pd ELSE 0 END) AS pd_pendientes
    -- (Aplica SUM similar para PE, PI y PP en afectadas y pendientes)
FROM config.catalogo_ejercicios e
INNER JOIN config.catalogo_periodos p ON e.eje_id = p.per_ejercicio_id
LEFT JOIN polizamovto m 
    ON p.per_ejercicio_id = m.ejercicio 
    AND p.per_mes = m.mes 
    AND m.estatus < 4
WHERE p.per_ejercicio_id = @p_ejercicio -- Parámetro de entrada
GROUP BY e.eje_anio, p.per_ejercicio_id, p.per_mes, p.per_estatus, p.per_enviado_contabilidad
ORDER BY p.per_mes ASC;

9 Reglas de Negocio Críticas y Límites de Validación
CódigoRegla / Límite de ValidaciónConsecuencia si no se cumpleDónde se valida
RN-CM-01 Pólizas Aplicadas: Validar que todas las pólizas del mes estén aplicadas (afecta = 1 o equivalente). No permite avanzar. Retorna mensaje: "Existen pólizas sin aplicar. Aplíquelas o cancélelas". SP fn_abc_cerrar_mes
RN-CM-02 Cuadre Contable: Validar que todas las pólizas del mes estén cuadradas matemáticamente (Σ Cargos = Σ Abonos). Detiene el cierre. Retorna mensaje para que el usuario analice, corrija la póliza o la cancele. SP fn_abc_cerrar_mes
RN-CM-03 Unidad Administrativa en Gasto: Validar que todas las pólizas de gasto tengan asignada explícitamente su Unidad Administrativa (Organización). Detiene el cierre. Exige que el usuario edite la póliza para colocar la unidad, o bien la cancele. SP fn_abc_cerrar_mes
RN-CM-04 Vigencia de Cuentas: Validar que todas las cuentas contables afectadas por las pólizas del mes se encuentren "Vigentes" en el Catálogo de Cuentas. Detiene el cierre. Solicita al usuario que corrija las cuentas en la póliza o cancele la operación. SP fn_abc_cerrar_mes
RN-CM-05 Secuencialidad de Meses: No se puede cerrar el mes N si el mes N-1 sigue abierto. Error en validación inicial: "El mes anterior no está cerrado". SP fn_abc_cerrar_mes
RN-CM-06 Reversibilidad Transaccional (Rollback): El proceso de cierre debe ejecutarse bajo una transacción (BEGIN TRY... CATCH). Si el mes se marca como cerrado pero una validación post-cierre detecta un error o inconsistencia, el sistema revierte el proceso. Se revierte (vuelve a abrir) y el mes se queda en estatus "Abierto / En Revisión". SP fn_abc_cerrar_mes
10 Flujos de Integración

Impacta transversalmente a los módulos operativos (Pólizas, Presupuesto, Compras). Si per_estatus == true para un mes en la tabla de catalogo_periodos, el sistema debe denegar el guardado de cualquier operación con fecha que caiga en ese mes.
