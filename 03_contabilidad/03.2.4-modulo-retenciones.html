03.2.4

Catálogo y Módulo de Retenciones

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV (Referencia: Manual PJEV 239).

Tipo: Catálogo Base Configurable
1 Descripción General

Este módulo permite configurar los conceptos de descuento que se aplican a los pagos, como el Impuesto Sobre la Renta (ISR), el Impuesto al Valor Agregado (IVA) retenido, cuotas al IPE u otras contribuciones. Vincula directamente cada concepto de retención con su cuenta contable pasiva correspondiente (2117), automatizando así las provisiones de pago de impuestos en el backend.

5 Stored Procedures (SPs)

Todas las interacciones de base de datos se encapsulan en las siguientes funciones de PostgreSQL:

  • fn_leer_cat_retenciones
    IN: @usuario, @buscar_texto, @page, @limit
    OUT: Recordset paginado.
  • fn_crud_cat_retenciones
    IN: @usuario, @accion, @ret_id, @descripcion, @porcentaje, @cta_pasivo_id
    OUT: {status}, {message}

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas:

  • Registro Nuevo: Permite ingresar el nombre del impuesto, la tasa y asociarlo a una cuenta del pasivo circulante mediante un buscador.
  • Desactivar Regla: Para impuestos que cambian de año (e.g. cambios de miscelánea fiscal).
  • Impresión / Exportación: Botón que exporta a PDF/Excel el "Catálogo Vigente de Retenciones", necesario para manuales de usuario del área de pagos.

10 Flujos de Integración

Módulo fuertemente integrado con Egresos y Tesorería. Cuando el operador captura un pasivo o emite un cheque y selecciona "Aplica Retención X", el sistema consulta esta tabla para saber matemáticamente cuánto descontar del subtotal y, más importante, a qué cuenta enviarlo en la póliza contable automática generada tras la emisión del pago.

PROCESO / ÁREA CONTABILIDAD SIAF-PJEV
1. CONFIGURACIÓN LEGAL Analiza las leyes fiscales y determina qué porcentaje de ISR, IVA o IPE se debe aplicar.
2. REGISTRO EN SISTEMA Captura la tasa y la asocia a una cuenta auxiliar de pasivo para la provisión. Verifica que la cuenta empiece con "2" (Pasivo) y la registra en base de datos.
3. DISPONIBILIDAD Inyecta estas opciones en el módulo de Egresos para que los analistas las seleccionen al crear pagos.
4. GENERACIÓN DE REPORTES Presiona el botón de imprimir para descargar el catálogo configurado. Exporta archivo PDF y Excel de soporte normativo.
