03.1.1

Alta de Cuentas Contables en el SAFPOJ

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV (Referencia: Manual PJEV 243).

Tipo: Catálogo Base
1 Descripción General

El módulo de Alta de Cuentas Contables permite a la Dirección de Contabilidad gestionar el Plan de Cuentas institucional (niveles 1 a 6). Asegura que el registro de nuevas cuentas, subcuentas y auxiliares (como nuevas cuentas bancarias o tipos de proveedores) se realice respetando la estructura jerárquica del CONAC, facilitando así el registro contable diario.

5 Stored Procedures (SPs) y Triggers

Las operaciones y reglas de negocio sobre el catálogo se gestionan a través de las siguientes funciones y disparadores de PostgreSQL:

Stored Procedures:

  • fn_leer_cat_cuentas
    IN: @usuario, @cc_nivel, @cc_padre_id, @page, @limit
    OUT: Recordset jerárquico (cc_id, cc_cuentapuntos, cc_descripcion, cc_naturaleza, etc.) y {total_count} para paginación.
  • fn_crud_cat_cuentas
    IN: @usuario, @accion (ALTA, MOD, BAJA), @cc_id, @cc_descripcion, @cc_tipocuenta, @cc_naturaleza, @cc_padre_id, @cc_permite_poliza
    OUT: {status}, {message}, {new_cc_id}
  • fn_lista_cuentas_poliza
    IN: @p_texto, @p_limit, @p_offset
    OUT: JSON jerárquico con coincidencias por texto o código. Exclusivo para búsqueda rápida en módulo de pólizas (solo retorna cuentas cc_tipocuenta = 'R' y cc_permite_poliza = 1).

Triggers Activos:

  • trg_calc_cuenta_clave: Se ejecuta ANTES de INSERT o UPDATE. Invoca fn_calc_cuenta_clave() para recalcular automáticamente los campos cc_cuentacompleta y cc_cuentapuntos en base a los niveles padre.
  • trg_protect_conac_delete / trg_protect_conac_update: Se ejecutan ANTES de DELETE/UPDATE. Invocan fn_protect_conac() para impedir modificaciones estructurales si cc_nivel <= 3 (Cuentas blindadas del CONAC).

7 Acciones del Módulo

El sistema expone 3 acciones críticas operadas desde la interfaz:

  • Registro Nuevo (Crear Nodo): Agrega una cuenta hija dependiendo del nodo seleccionado. El sistema valida que el padre exista y hereda la naturaleza.
  • Inactivación: Oculta la cuenta del seleccionador de pólizas. Valida que su saldo actual sea cero.
  • Impresión / Exportación: Botón que permite descargar el Plan de Cuentas completo en formato PDF o Excel, obligatorio para anexar a reportes de control.

9 Reglas de Negocio Críticas
CódigoReglaConsecuenciaDónde se validaFuente (Normativa/Técnica)
RN-CT-01 Inmutabilidad de Edición (Cuentas Creadas): Una vez creada la cuenta, los campos de clave jerárquica (Niveles 1-6), tipo, naturaleza, padre y legacy quedan INMUTABLES. El frontend deshabilita los inputs (`disabled`). Solo permite modificar: Descripción, Nivel de Afectación y Estatus. Frontend (page.tsx) y SP (`fn_crud_cat_cuentas`) Control Interno (Auditoría de Saldos)
RN-CT-02 Protección CONAC (Niveles Base): Las cuentas de Nivel 1, 2 y 3 están blindadas. Despliega aviso en UI: "RN-09 — Cuenta CONAC... campos bloqueados". Impide eliminación o alteración estructural desde BD. Frontend y Trigger (`trg_protect_conac`) LGCG Plan de Cuentas
RN-CT-03 Coherencia de Nivel de Afectación: Si la cuenta es "Agrupadora (A)", el nivel obligatoriamente es 0. Si es "Registro (R)", puede ser 0:CUENTA, 1:EMPLEADOS, 2:PROVEEDORES, 3:UNIDAD ADMINISTRATIVA, 4:TERCEROS BENEFICIARIOS, 5:TERCEROS INSTITUCIONALES, 6:BIENES INMUEBLES. El select restringe dinámicamente las opciones. Permite a los módulos de pólizas exigir subcuentas (Ej. Proveedor específico) según el valor. Frontend y BD (`chk_nivelafecta_tipocuenta`) Regla de Sumarización y Pólizas
RN-CT-04 Baja Lógica (Soft Delete): "Eliminar" una cuenta no la borra de la BD, solo cambia su `estatus_registro` a 0 (Inactiva). Protege la integridad referencial de pólizas pasadas. Despliega modal de confirmación en UI advirtiendo conservación de auditoría. Frontend (Modal) y API (DELETE logic) LGCG (Integridad Histórica)
RN-CT-05 Relación Excluyente CRI / COG: Solo pueden habilitarse en los niveles 5 y 6. Solo puedes relacionar UNA cuenta CRI o UNA cuenta COG a un auxiliar, NUNCA ambas. El sistema deshabilita una opción si la otra es llenada, o marca error de validación cruzada. Obliga a que la cuenta funcione estrictamente para Ingresos o para Egresos conforme a la matriz de conversión. Frontend y Validaciones Backend Normatividad CONAC (Matrices de Conversión)
10 Flujos de Integración

El catálogo de cuentas contables es el corazón del sistema SAFPOJ. Hacia atrás, no depende de ningún otro módulo, es el origen de los datos maestros. Hacia adelante, alimenta a la Matriz de Conversión (Ingresos/Egresos), a los Módulos de Pólizas, Fideicomisos y es la base estructural para la Emisión de Estados Financieros.

PROCESO / ÁREA CONTADOR / ANALISTA SIAF-PJEV
1. IDENTIFICACIÓN NECESIDAD Detecta la necesidad de una cuenta auxiliar para registrar movimientos bancarios o nuevos deudores.
2. REGISTRO EN SISTEMA Ingresa al módulo, selecciona el nodo padre (Nivel 4 o superior) y teclea la descripción del auxiliar. Despliega el árbol contable para asegurar que la jerarquía base sea la correcta.
3. VALIDACIÓN TÉCNICA Valida las reglas RN-CT-01 (protección CONAC) y RN-CT-02 (herencia de naturaleza) y ejecuta la persistencia.
4. GENERACIÓN DE REPORTES Presiona el botón de imprimir para descargar el plan de cuentas y notificar a su jefatura. Exporta el PDF oficial conteniendo el árbol jerárquico actualizado.
