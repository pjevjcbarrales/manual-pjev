03.1.2

Matriz de Conversión (Ingresos y Egresos)

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV.

Tipo: Catálogo Base Configurable
1 Descripción General

La Matriz de Conversión es el cerebro del "Registro Único". Es un diccionario de reglas que traduce cada transacción presupuestal (COG) a su equivalente contable (Cuenta). Cuando el área de Egresos realiza el pago de una factura (Partida 21101), la matriz indica al sistema que automáticamente debe cargar a Gastos (5121) y abonar a Bancos (1112).

5 Stored Procedures (SPs)

Todas las interacciones de base de datos se encapsulan en las siguientes funciones de PostgreSQL:

  • fn_leer_matriz_conversion
    IN: @usuario, @tipo_clasificador, @page, @limit
    OUT: Recordset con los mapeos actuales COG->Cuentas.
  • fn_crud_matriz_conversion
    IN: @usuario, @accion, @mtrz_id, @clasificador_id, @tipo_movimiento, @cta_cargo, @cta_abono
    OUT: {status}, {message}

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas:

  • Mapeo Rápido: Permite seleccionar una partida presupuestal y mediante un formulario asignar la cuenta de cargo y abono para los momentos contables.
  • Clonación de Reglas: Permite copiar la configuración de un Ejercicio Fiscal anterior al actual para evitar recapturas masivas en Enero.
  • Impresión / Exportación: Acción obligatoria para exportar a PDF/Excel la "Matriz de Conversión Vigente", comúnmente solicitada en auditorías de sistemas.

10 Flujos de Integración

La Matriz interactúa masivamente de forma invisible en el backend. Cuando el Módulo de Control de Gastos (Tesorería) emite una orden de pago y aprueba el egreso, el sistema de pólizas lee las partidas presupuestales de esa orden de pago y hace un JOIN con esta matriz para generar instantáneamente los renglones de la póliza de diario con las cuentas CTA_CARGO y CTA_ABONO configuradas aquí.

11 Afectación Contable (Reglas CONAC)
AplicaDescripción Contable

PROCESO / ÁREA CONTABILIDAD SIAF-PJEV
1. IDENTIFICACIÓN Al emitirse un nuevo clasificador COG, identifica las cuentas contables patrimoniales correspondientes a cargar y abonar.
2. CONFIGURACIÓN Carga las reglas en el módulo de Matriz. Valida que las cuentas ingresadas sean afectables (nivel mínimo 5) y las guarda.
3. OPERACIÓN AUTOMÁTICA Cuando Presupuestos ejerce un pago, el Motor de Pólizas consulta silenciosamente la matriz y genera la póliza cuadrada sin intervención humana.
4. AUDITORÍA Presiona el botón de imprimir para descargar la Matriz de Conversión Vigente. Construye el archivo PDF/Excel con la relación COG-Cuentas.
