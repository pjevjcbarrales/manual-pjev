03.3.2

Conciliación Bancaria Contable

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV.

Tipo: Herramienta de Auditoría y Cuadre
1 Descripción General

Este módulo compara los registros financieros del sistema (Cuenta Contable 1112 - Bancos) contra el extracto o estado de cuenta emitido por la institución bancaria (archivo TXT/CSV/PDF). Su objetivo es identificar "partidas en tránsito" (cheques no cobrados) o comisiones bancarias no registradas en el sistema para que Contabilidad y Tesorería mantengan el saldo real y el saldo en libros 100% igualados.

Definición SQL (PostgreSQL)

-- ==========================================
-- 1. LEER MOVTOS PLAZAS
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_leer_movtos_plazas(
    p_idbanco INT,
    p_fechaini DATE,
    p_fechafin DATE
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    -- Lógica para extraer cuenta contable del banco y movimientos
    SELECT COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb) INTO v_resultado
    FROM (
        SELECT p.folio_poliza AS poliza, p.fecha_poliza, m.importe,
               COALESCE(cb.conciliado, 0) AS conciliado
        FROM contabilidad.polizas p
        JOIN contabilidad.polizas_movtos m ON p.id_poliza = m.id_poliza
        LEFT JOIN contabilidad.conciliacion_bancaria cb ON p.id_poliza = cb.id_poliza AND cb.id_banco = p_idbanco
        WHERE m.cuenta_contable = (SELECT cuenta_contable FROM bancos WHERE id_banco = p_idbanco)
          AND p.fecha_poliza BETWEEN p_fechaini AND p_fechafin
        ORDER BY p.fecha_poliza, p.folio_poliza
    ) t;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

@p_id_polizaINTID de la póliza.

Valores Devueltos (OUT) JSONTipoDescripción

Definición SQL (PostgreSQL)

-- ==========================================
-- 2. ABC MOVTOS
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.sp_abc_movtos(
    p_id_poliza INT,
    p_id_banco INT,
    p_conciliado INT
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_existe INT;
BEGIN
    SELECT 1 INTO v_existe FROM contabilidad.conciliacion_bancaria 
    WHERE id_poliza = p_id_poliza AND id_banco = p_id_banco;
    
    IF v_existe = 1 THEN
        UPDATE contabilidad.conciliacion_bancaria
        SET conciliado = p_conciliado,
            fecha_conciliacion = CURRENT_DATE,
            fecha_m = CURRENT_TIMESTAMP
        WHERE id_poliza = p_id_poliza AND id_banco = p_id_banco;
    ELSE
        INSERT INTO contabilidad.conciliacion_bancaria (
            id_poliza, id_banco, conciliado, fecha_conciliacion, estatus_registro, fecha_c
        ) VALUES (
            p_id_poliza, p_id_banco, p_conciliado, CURRENT_DATE, 1, CURRENT_TIMESTAMP
        );
    END IF;
    
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'Guardado con éxito');
END;
$$;

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas:

  • Sumatoria y Diferencia: El sistema presenta la sumatoria del Saldo en Libros. El usuario debe ingresar manualmente el Saldo en Bancos (obtenido de su estado de cuenta impreso) para que el sistema calcule automáticamente la diferencia.
  • Lectura de Movimientos: Llama a fn_leer_movtos_plazas para llenar la grilla con las pólizas y sus importes, mostrando la bandera si ya están conciliadas.
  • Marcar Conciliado: Permite al usuario seleccionar el movimiento (checkbox o botón) para cambiar su estatus a "Conciliado" [1] o revertirlo [0].
  • Guardar (Guardado Masivo/Batch): Al presionar Guardar, se ejecuta el proceso (sp_abc_movtos) iterando por todos los movimientos modificados. Esto permite determinar el avance del proceso, validando si se actualiza o inserta en la tabla puente, y se presenta una barra/porcentaje de avance al usuario en la interfaz para que vea el progreso del guardado masivo.
  • Impresión / Exportación: Botón que genera la hoja oficial de "Conciliación Bancaria".

10 Flujos de Integración

Es un proceso de validación. Consume los movimientos de las pólizas que tocaron la cuenta 1112 (Bancos). Si durante la conciliación se detectan intereses bancarios a favor o cobro de comisiones que el sistema ignora, el contador debe ir al módulo de Pólizas Manuales (03.2.2), capturar el gasto financiero y volver a este módulo para matchearlo, logrando el cuadre.

PROCESO / ÁREA CONTABILIDAD SIAF-PJEV
1. CARGA INICIAL Ingresa al portal del Banco y descarga el estado de cuenta. Lo sube al SIAF. Carga el archivo, suma los importes y activa el algoritmo de Auto-Match buscando referencias idénticas.
2. DEPURACIÓN MANUAL Arrastra las partidas no cruzadas en pantalla. Si faltan pólizas (comisiones), detiene el proceso, va a Pólizas Manuales, las captura y vuelve. Actualiza la fórmula de "Diferencia" en tiempo real conforme el usuario cruza partidas.
3. CIERRE DEL MES Una vez que la Diferencia es matemáticamente cero, presiona el botón "Cerrar Mes". Cambia el estatus a 3. Bloquea la cuenta para impedir que alguien inserte pólizas con fechas pasadas en ese banco.
4. GENERACIÓN DE REPORTES Presiona Imprimir para recabar firmas del Tesorero y Director. Exporta la hoja de conciliación con los anexos de Cheques en Tránsito.
