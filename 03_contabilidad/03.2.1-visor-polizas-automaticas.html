03.2.1

Visor de Pólizas Automáticas

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV (Referencia: Manual PJEV 246).

Tipo: Módulo de Registro Transaccional
1 Descripción General

Este módulo es la bandeja de entrada del Jefe de Contabilidad. Cuando el área de Presupuesto o Tesorería ejecuta una operación patrimonial, el sistema genera de forma invisible (basado en la matriz de conversión) una póliza contable automática. Este visor permite al contador revisarla, validar su cuadratura y presionar "Afectar Póliza", lo cual la inyecta definitivamente a los libros de mayor bloqueando el documento origen (candado bidireccional).

5 Stored Procedures (SPs)

Todas las interacciones de base de datos se encapsulan en las siguientes funciones de PostgreSQL:

  • fn_leer_polizas_automaticas
    IN: @usuario, @fecha_inicio, @fecha_fin, @estatus, @page, @limit
    OUT: Recordset con cabeceras de pólizas.
  • fn_cambiar_estatus_poliza
    IN: @usuario, @pol_id, @nuevo_estatus (AFECTAR/DESAFECTAR)
    OUT: {status}, {message}

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas (No existe alta porque se crean solas):

  • Afectación Masiva: Seleccionar N pólizas y cambiar su estatus a 2 (Afectada).
  • Desafectación: Libera la póliza (y el documento origen) bajando su estatus a 1. Requiere privilegios elevados.
  • Visualización de Detalles: Abre una tabla de solo lectura mostrando los asientos de Cargo y Abono (Debe/Haber).
  • Impresión / Exportación: Botón 🖨️ para imprimir la "Póliza Contable Formato CONAC" que será firmada por el Contador General.

10 Flujos de Integración

Es el gran colector de datos de todo el SIAF-PJEV. Recibe información de Autorización Presupuestal, Órdenes de Pago, Viáticos, y Recibos de Ingresos. Hacia adelante, las pólizas con estatus 2 alimentan matemáticamente a la Balanza de Comprobación y, por ende, a los Estados Financieros.

PROCESO / ÁREA CONTABILIDAD ÁREAS EXTERNAS (PPTO/TESORERÍA) SIAF-PJEV
1. ORIGEN TRANSACCIÓN Ejecutan su flujo normal (Ej. Orden de Pago). Intercepta la operación y genera póliza de forma asíncrona.
2. REVISIÓN CONTABLE Ingresa al visor diario, identifica pólizas "Por Afectar" y revisa los asientos Debe/Haber generados. Muestra grid bloqueado (solo lectura).
3. AFECTACIÓN Y BLOQUEO Presiona el botón "✓ Afectar". Recibe bloqueo en sistema. Ya no puede editar ni borrar la orden de pago original. Aplica Estatus 2, suma a Balanza, e implementa Candado Bidireccional.
4. GENERACIÓN DE REPORTES Presiona el botón de imprimir para archivar físicamente la póliza o subir XML a transparencia. Genera archivo PDF en formato CONAC y Excel.
