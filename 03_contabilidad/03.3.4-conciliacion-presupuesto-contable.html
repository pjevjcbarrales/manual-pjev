03.3.4

Conciliación de Egresos Presupuestarios vs Gastos Contables

Análisis Funcional bajo el Estándar Oficial DOC-08 del SIAF-PJEV (Referencia: Manual PJEV 278).

Tipo: Herramienta de Auditoría Estructural (ORFIS)
1 Descripción General

Este módulo es la máxima prueba de fuego de la Matriz de Conversión. Compara que la suma total del dinero gastado según Presupuestos (Egresos Pagados - Cuentas 82X0) sea exactamente igual a la suma del Gasto según Contabilidad (Cuentas 5XXX). Existen partidas presupuestales que NO son gasto contable (ej. comprar un coche es Activo Fijo, no gasto) y gastos contables que NO son egreso presupuestal (ej. la depreciación). El sistema genera la ecuación matemática automática para justificar esas diferencias legales.

Definición SQL (PostgreSQL)

-- ==========================================
-- 1. CONCILIACIÓN EGRESOS VS GASTOS
-- ==========================================
CREATE OR REPLACE FUNCTION contabilidad.fn_generar_conciliacion_egresos_gastos(
    p_ejercicio INT,
    p_mes_corte INT
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    SELECT row_to_json(t) INTO v_resultado
    FROM (
        SELECT 
            15000000.00 AS "Total_Egresos_Presupuestarios",
            500000.00 AS "Egresos_No_Contables",
            200000.00 AS "Gastos_No_Presupuestarios",
            14700000.00 AS "Total_Gasto_Contable",
            0.00 AS "Diferencia_Matematica"
        -- Lógica real: sumatorias de tablas de saldos filtradas por cuentas específicas
    ) t;
    
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

7 Acciones del Módulo

El sistema expone las siguientes acciones operativas:

  • Cálculo en Tiempo Real: El usuario selecciona un periodo y el motor procesa miles de pólizas al instante para generar el anexo.
  • Drill-down (Exploración): Si hay un descuadre (Diferencia != 0), el usuario puede hacer clic en los rubros para ver las pólizas exactas que conforman esa bolsa de dinero, facilitando cazar el error humano.
  • Impresión / Exportación: Botón que exporta el Anexo CONAC obligatorio para la Cuenta Pública.

10 Flujos de Integración

Es la culminación de todo el engranaje del SIAF-PJEV. Reúne la ejecución del Módulo de Egresos, lo cruza por la Matriz de Conversión, incluye las facturas de la Bóveda CFDI, las amortizaciones de la Descarga Masiva y los ajustes de las Pólizas Manuales, condensando miles de transacciones en 5 líneas matemáticas que demuestran que el sistema no "pierde" centavos.

PROCESO / ÁREA CONTABILIDAD SIAF-PJEV
1. SOLICITUD DE REPORTE Al cerrar trimestre, ingresa la fecha límite y presiona calcular. Ejecuta la macro consulta SQL sumando todos los egresos presupuestales y aislando el activo fijo.
2. REVISIÓN DE INTEGRIDAD Si nota "Diferencia", hace drill-down y averigua si una póliza manual se capturó mal (Ej. Cargo a Gasto sin abonar a presupuesto). Cancela la póliza y la corrige. Permite desglose de las operaciones (drill-down view).
3. GENERACIÓN DE REPORTES Con Diferencia Cero, presiona Imprimir. Exporta el PDF Oficial requerido por ORFIS validando la solidez de la Matriz de Conversión.
