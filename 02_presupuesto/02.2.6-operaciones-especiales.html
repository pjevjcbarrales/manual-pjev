02.2.6

Operaciones Especiales Presupuestales

Consola centralizada para ejecutar movimientos y ajustes contables-presupuestarios extraordinarios requeridos por el área de Presupuesto, abarcando tanto afectaciones globales de Egresos como pólizas iniciales de Ingresos.

Tipo: Herramientas de Administración Presupuestal
1 Descripción General

Este módulo actúa como una consola de administración que consolida rutinas masivas y operaciones especiales posteriores a la formulación del presupuesto. Su objetivo es automatizar procesos manuales complejos, aislando la lógica de negocio contable de los procesos transaccionales del día a día.

4 Operaciones de Egresos

Ajustes masivos y reclasificaciones sobre el presupuesto de egresos (Techo Presupuestal). Incluye operaciones críticas como:

  • Compromiso Presupuestal del Capítulo 1000: Automatización requerida para mover contablemente la totalidad del saldo del Capítulo 1000 (Servicios Personales) hacia el momento presupuestario de Comprometido. Se realiza mediante la generación y afectación de una Póliza Presupuestal masiva. Esto permite reservar el presupuesto de nómina de manera inmediata a inicios de ejercicio fiscal.
5 Operaciones de Ingresos

Tratamiento especial de la Ley de Ingresos, enfocado exclusivamente en las cifras originales y sus modificaciones, dejando fuera la transaccionalidad de recaudación. Importante: De acuerdo a la normativa de balance presupuestario, la ejecución de las operaciones de ingresos parte y debe encontrarse estrictamente fundamentada en la correcta captura de las operaciones de egresos correspondientes para mantener el equilibrio contable. Incluye:

  • Póliza de Apertura de Ingresos: Generación de la póliza contable automática que registra el estimado original (Apertura) al arrancar el ejercicio fiscal, impactando las cuentas de orden de Ingresos Estimados.
  • Movimientos del Presupuesto de Ingresos: Interfaz para el registro manual y afectación de ampliaciones y reducciones sobre la Ley de Ingresos Estimada o Modificada (excluyendo la fase del Recaudado/Devengado, la cual pertenece enteramente a operaciones transaccionales o de Tesorería).

7 Funciones y Procedimientos (Backend)

Cada acción desencadenada desde los botones de la consola ejecuta procedimientos específicos en la base de datos para generar las pólizas y afectaciones automáticas. Nota de Seguridad: Toda comunicación con los endpoints asociados a estas funciones debe estar implementada y documentada en Swagger, y el acceso debe ser estrictamente protegido validando el token JWT.

Calcula el saldo total del Capítulo 1000 y genera la póliza presupuestal masiva que reclasifica el monto del Aprobado/Modificado al momento Comprometido.

Snippet PostgreSQL

CREATE OR REPLACE FUNCTION presupuesto.fn_generar_compromiso_capitulo_1000(
  p_accion VARCHAR, p_usuario VARCHAR, p_ejercicio INT,
  p_ip VARCHAR, p_mac VARCHAR
) RETURNS JSON AS $$
DECLARE
  v_res JSON;
  v_folio VARCHAR;
BEGIN
  -- 1. Validar que no se haya comprometido previamente
  -- 2. Sumarizar saldos disponibles del Capítulo 1000
  -- 3. Generar la póliza presupuestal de Egresos
  -- 4. Afectar cuentas de orden (Aprobado/Modificado -> Comprometido)
  
  v_res := json_build_object('estatus', 1, 'msg_resultado', 'Póliza generada', 'data', json_build_object('folio', v_folio));
  RETURN v_res;
EXCEPTION WHEN OTHERS THEN
  RETURN json_build_object('estatus', 0, 'msg_resultado', SQLERRM, 'data', null);
END;
$$ LANGUAGE plpgsql;

CREATE OR REPLACE FUNCTION presupuesto.fn_genera_movimiento_ingreso_modificacion(
  p_accion VARCHAR, p_usuario VARCHAR, p_ejercicio INT,
  p_tipo_movimiento VARCHAR, p_fecha_movimiento DATE, p_justificacion TEXT,
  p_detalles JSON, p_ip VARCHAR, p_mac VARCHAR
) RETURNS JSON AS $$
DECLARE
  v_res JSON;
  v_mod_id BIGINT;
  v_folio VARCHAR;
BEGIN
  -- 1. Generar encabezado del ajuste a la Ley de Ingresos
  -- 2. Insertar CRI afectados iterando p_detalles
  -- 3. Detonar póliza concurrente que ajusta el Estimado/Modificado
  -- 4. Validar el balance presupuestario (Ingresos vs Egresos)

  RETURN v_res;
