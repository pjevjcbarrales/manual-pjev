02.1.1

Catálogo de Clave Presupuestal

Maestro de clasificación presupuestal del sistema. Define todas las combinaciones válidas de clasificadores que identifican un gasto y sirve como referencia para el Presupuesto de Egresos.

Tipo: Catálogo Maestro
1 Descripción General

El Catálogo de Clave Presupuestal organiza sus registros en 6 fuentes de financiamiento. Cada registro corresponde a una línea presupuestal de un ejercicio fiscal. A diferencia de otros módulos, la tabla presupuesto.clave_presupuestal no tiene modelo Prisma, y su acceso se hace directamente con $queryRaw a través de funciones y stored procedures de PostgreSQL para soportar particionamiento estricto por ejercicio.

7 Acciones del Módulo
  • Registro (CRUD): Botón [+ NUEVO REGISTRO] que abre una modal con un formulario completo para armar una clave a mano. En el grid, cada clave tiene acciones [✏ EDITAR] y [🗑 ELIMINAR].
  • Especiales: Botón [📤 IMPORTAR ARCHIVO] que abre un stepper. Paso 1: Selecciona Excel. Paso 2: El sistema verifica (endpoint `/validar-clave`) y muestra errores. Paso 3: Confirmación de borrado masivo e inserción.
  • Vacío: N/A
  • Reportes: Botón [🖨 IMPRIMIR CATÁLOGO] para exportar el grid filtrado.

10 Flujos de Integración

Este catálogo es la piedra angular paramétrica. El módulo de Presupuesto Autorizado hace un JOIN directo para obtener la configuración. Notablemente, el sistema permite cruces multianuales sin bloquear claves de años previos para consultas históricas.

11 Afectación Contable (Reglas CONAC)
AplicaDescripción Contable
NO La creación de la Clave Presupuestal es un proceso de diccionario paramétrico. No implica asignación de montos ni transacciones financieras, por lo que no detona asientos contables ni afecta saldos. El ciclo contable inicia hasta el registro del "Presupuesto de Egresos Aprobado" en el módulo subsecuente.

PROCESO / ÁREA PRESUPUESTOS SIAF-PJEV
1. IDENTIFICACIÓN DE ESTRUCTURA Identifica clasificadores presupuestales según lineamientos del CONAC y elabora la matriz de claves para el ejercicio fiscal entrante.
2. CARGA O ALTA DE CLAVES Realiza la importación masiva mediante archivo de Excel o la captura individual de registros en el sistema. Recibe petición de carga y lee datos del archivo o formulario.
3. VALIDACIÓN PARAMÉTRICA Si el sistema arroja errores críticos (ej. partidas inexistentes), corrige el archivo de carga y vuelve a intentar. Ejecuta Stored Procedure de Validación. Cruza datos contra catálogos base (UA, COG, Fondo). Emite reporte de errores si los hay.
4. REGISTRO Y VINCULACIÓN CONTABLE Guarda registros en BD particionada, asigna sector fijo (301-C) y vincula automáticamente la partida presupuestal con su Plan de Cuentas (Regla CONAC).
5. APERTURA DE EJERCICIO Imprime catálogo final, archiva documento soporte e informa la habilitación de las claves para el inicio del Presupuesto Autorizado.
