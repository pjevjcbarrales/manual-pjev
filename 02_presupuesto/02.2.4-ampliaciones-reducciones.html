02.2.4

Amp. y Red. (Egresos)

Módulo transaccional para incrementar o disminuir el Techo Presupuestal de Egresos. Opera con base en oficios notificatorios (ej. SEFIPLAN) y genera afectaciones directas al Presupuesto Modificado, alterando las bolsas del presupuesto disponible.

Tipo: Movimiento Presupuestal Directo
1 Descripción General

Este procedimiento registra las inyecciones de liquidez adicional (Ampliaciones) o los recortes presupuestales (Reducciones) dictados por una autoridad competente, típicamente la SEFIPLAN. A diferencia de las Transferencias Compensadas (que suman 0 al moverse entre partidas), estos movimientos alteran el total global del Presupuesto de Egresos del ejercicio en curso. Toda operación exige adjuntar la evidencia documental (Oficio) y asigna un folio consecutivo.

CREATE OR REPLACE FUNCTION presupuesto.fn_abc_movimiento_modificacion(
  p_accion VARCHAR, p_usuario VARCHAR, p_ip VARCHAR, p_mac VARCHAR,
  p_mod_id BIGINT, p_tipo_movimiento VARCHAR, p_folio VARCHAR,
  p_oficio VARCHAR, p_fecha_aplicacion DATE, p_estatus VARCHAR,
  p_observaciones TEXT, p_estatus_registro SMALLINT
) RETURNS JSON AS $$
DECLARE
  v_res JSON;
BEGIN
  IF p_accion = 'A' THEN
    INSERT INTO presupuesto.movimientos_modificacion (
      tipo_movimiento, folio, oficio, fecha_aplicacion, estatus, observaciones, estatus_registro, usuario_c
    ) VALUES (
      p_tipo_movimiento, p_folio, p_oficio, p_fecha_aplicacion, COALESCE(p_estatus, 'APLICADO'), p_observaciones, COALESCE(p_estatus_registro, 1), p_usuario
    ) RETURNING mod_id INTO p_mod_id;
    v_res := json_build_object('estatus', 1, 'msg_resultado', 'Guardado correctamente', 'data', json_build_object('id', p_mod_id));
  ELSIF p_accion = 'M' THEN
    UPDATE presupuesto.movimientos_modificacion SET
      tipo_movimiento = p_tipo_movimiento, folio = p_folio, oficio = p_oficio,
      fecha_aplicacion = p_fecha_aplicacion, estatus = p_estatus, observaciones = p_observaciones, estatus_registro = p_estatus_registro
    WHERE mod_id = p_mod_id;
    v_res := json_build_object('estatus', 1, 'msg_resultado', 'Actualizado correctamente', 'data', json_build_object('id', p_mod_id));
  ELSIF p_accion = 'B' THEN
    UPDATE presupuesto.movimientos_modificacion SET estatus_registro = 0 WHERE mod_id = p_mod_id;
    v_res := json_build_object('estatus', 1, 'msg_resultado', 'Eliminado correctamente', 'data', json_build_object('id', p_mod_id));
  END IF;
  RETURN v_res;
EXCEPTION WHEN OTHERS THEN
  RETURN json_build_object('estatus', 0, 'msg_resultado', SQLERRM, 'data', null);
END;
$$ LANGUAGE plpgsql;

  v_data JSON;

      FROM presupuesto.movimientos_modificacion WHERE estatus_registro = 1

    SELECT row_to_json(t) INTO v_data FROM (

      FROM presupuesto.movimientos_modificacion WHERE mod_id = p_mod_id

  RETURN json_build_object('estatus', 0, 'msg_resultado', SQLERRM, 'data', null);
END;

10 Flujos de Integración

El Módulo de Ampliaciones y Reducciones no opera de manera aislada; interactúa directamente con los siguientes módulos del ecosistema SIAF-PJEV:

  • Contabilidad Gubernamental: Detona la generación de pólizas automáticas en las cuentas de orden de presupuesto.
  • Control Presupuestal (Disponibilidad): Todo movimiento autorizado altera en tiempo real la sumatoria de Egresos por Ejercer, afectando las validaciones de suficiencia para las requisiciones y órdenes de compra.
  • Tesorería: Tesorería recibe notificación (vía tableros o reportes operativos) del impacto presupuestal para considerar la viabilidad de los flujos de efectivo.
11 Afectación Contable (Reglas CONAC)

Todo movimiento impacta la cuenta de Modificaciones, la cual altera indirectamente la bolsa de Egresos por Ejercer en las Cuentas de Orden (Serie 8000).

PROCESO / ÁREA PRESUPUESTO (DPP) TESORERÍA SIAF-PJEV
1. IDENTIFICACIÓN Y VERIFICACIÓN Analiza Oficio de SEFIPLAN, ingresa al SIAF para validar que la Clave Presupuestal exista. Valida la base de datos de claves. Si no existe, requiere flujo alterno (Tarjeta Informativa).
2. CAPTURA Y VALIDACIÓN Ingresa folio, anexa PDF y especifica monto a Ampliar o Reducir. Si es reducción, verifica que no rebase el "Saldo Disponible" (Comprometido + Devengado).
3. AUTORIZACIÓN Y PÓLIZA Firma y Autoriza el movimiento. Actualiza cuentas 8.2.3 y 8.2.2 en contabilidad, y suma/resta el monto en el control presupuestal.
4. NOTIFICACIÓN Imprime oficio interno y envía a Tesorería. Recibe oficio para tener certeza de fondos antes de liberar pagos. Mantiene la bitácora de auditoría inalterable ligada al Folio SEFIPLAN.
