04.2.2

Registro de Ingresos (Devengado)

Módulo transaccional para el registro del Oficio de SEFIPLAN que notifica la transferencia de recursos al Poder Judicial.

Modo A — existe en código Tipo: Análisis Funcional (DOC-08)
1 Descripción General

Este submódulo atiende exclusivamente la fase uno (Tiempo 1) de la captación de recursos. Su objetivo es registrar formalmente en el SIAF-PJEV que se ha recibido una notificación de cobro a favor de la Institución.

El analista ingresa los datos del oficio (folio, fecha, importe y concepto). Al autorizar el registro, nace una Cuenta por Cobrar en la contabilidad y se devenga la Ley de Ingresos, quedando pendiente de cobro hasta que el área de Caja opere el módulo 04.2.3 Registro de Ingresos (Recaudado).

2 Marco Normativo
Norma Obligación que genera
Ley General de Contabilidad Gubernamental (LGCG) Obliga a registrar el Ingreso Devengado en el momento en que existe jurídicamente el derecho de cobro o se notifica oficialmente la transferencia (oficio).
Norma CONAC Establece que todo ingreso devengado debe reconocerse mediante un Cargo a una Cuenta por Cobrar a Corto Plazo (1.1.2.4) contra un Abono a Ingresos Patrimoniales.
3 Roles que intervienen
Rol Acción permitida Restricción
Jefe de Presupuesto Genera el Oficio de solicitud de recursos a SEFIPLAN y lo turna a Tesorería para la captura del devengo. Genera la orden documental, pero no realiza la afectación contable directa en el sistema.
Analista de Ingresos (Tesorería) Recibe el Oficio de Presupuestos y captura el registro de la Ministración esperada. La captura se guarda en estatus "Borrador". No genera póliza.
Contabilidad (Subdirector) Revisa y Autoriza el oficio capturado. Al autorizar, el registro se bloquea y pasa al módulo de Recaudación (Caja).

Definición SQL (PostgreSQL)

-- ==========================================
-- 1. LEER ENCABEZADO
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_leer_ingresos_devengo_encabezado(
    p_ejercicio INT,
    p_ingreso_id INT DEFAULT NULL,
    p_fecha_inicio DATE DEFAULT NULL,
    p_fecha_fin DATE DEFAULT NULL,
    p_estatus_documento INT DEFAULT NULL,
    p_oficio_sefiplan VARCHAR DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    SELECT COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb) INTO v_resultado
    FROM (
        SELECT 
            m.ingreso_id, m.folio_documento, m.fecha_documento, m.monto_total, m.estatus_documento
        FROM ingresos.ing_ingresos m
        WHERE m.ejercicio = p_ejercicio AND m.estatus_registro = 1 AND m.estatus_documento IN (1, 2)
          AND (p_ingreso_id IS NULL OR m.ingreso_id = p_ingreso_id)
          AND (p_fecha_inicio IS NULL OR m.fecha_documento >= p_fecha_inicio)
          AND (p_fecha_fin IS NULL OR m.fecha_documento <= p_fecha_fin)
          AND (p_estatus_documento IS NULL OR m.estatus_documento = p_estatus_documento)
          AND (p_oficio_sefiplan IS NULL OR m.folio_documento ILIKE '%' || p_oficio_sefiplan || '%')
    ) t;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;
Parámetros (IN)TipoDescripción
@p_ingreso_idINTObligatorio. ID del encabezado del oficio.
Valores Devueltos (OUT) JSONTipoDescripción
{ingreso_movto_id}INTID del detalle.
{centro_costo_id}INTID de Organización.
{concepto_id}INTID de Concepto de Ingreso.
{monto_organizacion}DECIMALMonto asignado al centro.
{concepto_desc}VARCHARDescripción del concepto.
{centro_costo_desc}VARCHARDescripción de la organización.

Definición SQL (PostgreSQL)

-- ==========================================
-- 2. LEER MOVTOS (DETALLE)
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_leer_ingresos_devengo_movtos(
    p_ingreso_id INT
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    SELECT COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb) INTO v_resultado
    FROM (
        SELECT 
            d.ingreso_movto_id, d.centro_costo_id, d.concepto_id, d.monto_organizacion,
            c.descripcion AS concepto_desc, o.descripcion AS centro_costo_desc
        FROM ingresos.ing_ingresos_movtos d
        LEFT JOIN ingresos.ing_conceptos c ON d.concepto_id = c.concepto_id
        LEFT JOIN configuracion.organizaciones o ON d.centro_costo_id = o.organizacion_id
        WHERE d.ingreso_id = p_ingreso_id AND d.estatus_registro = 1
    ) t;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

Definición SQL (PostgreSQL)

-- ==========================================
-- 3. ABC ENCABEZADO
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_abc_ingresos_devengo_encabezado(
    p_accion INT,
    p_ingreso_id INT DEFAULT NULL,
    p_folio_documento VARCHAR DEFAULT NULL,
    p_fecha_documento DATE DEFAULT NULL,
    p_monto_total DECIMAL DEFAULT NULL,
    p_usuario VARCHAR DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_id INT; v_poliza_id INT;
BEGIN
    IF p_accion = 1 THEN -- Borrador
        IF p_ingreso_id IS NULL OR p_ingreso_id = 0 THEN
            INSERT INTO ingresos.ing_ingresos (ejercicio, folio_documento, fecha_documento, monto_total, estatus_documento, estatus_registro, usuario_c, fecha_c)
            VALUES (EXTRACT(YEAR FROM p_fecha_documento), p_folio_documento, p_fecha_documento, p_monto_total, 1, 1, p_usuario, CURRENT_TIMESTAMP)
            RETURNING ingreso_id INTO v_id;
        ELSE
            UPDATE ingresos.ing_ingresos SET folio_documento = p_folio_documento, fecha_documento = p_fecha_documento, monto_total = p_monto_total, usuario_m = p_usuario, fecha_m = CURRENT_TIMESTAMP
            WHERE ingreso_id = p_ingreso_id AND estatus_documento = 1;
            v_id := p_ingreso_id;
        END IF;
    ELSIF p_accion = 2 THEN -- Autorizar
        UPDATE ingresos.ing_ingresos SET estatus_documento = 2, usuario_m = p_usuario, fecha_m = CURRENT_TIMESTAMP
        WHERE ingreso_id = p_ingreso_id AND estatus_documento = 1;
        v_id := p_ingreso_id;
    ELSIF p_accion = 3 THEN -- Cancelar
        UPDATE ingresos.ing_ingresos SET estatus_registro = 0, usuario_m = p_usuario, fecha_m = CURRENT_TIMESTAMP WHERE ingreso_id = p_ingreso_id;
        v_id := p_ingreso_id;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'id', v_id);
END;
$$;
Parámetros (IN)TipoDescripción
@p_accionINT1: Insertar/Actualizar, 3: Eliminar detalle.
@p_ingreso_movto_idINTNulo para insertar.
@p_ingreso_idINTID del oficio padre.
@p_centro_costo_idINTOrganización.
@p_concepto_idINTConcepto de Ingreso.
@p_monto_organizacionDECIMALMonto parcial.

Definición SQL (PostgreSQL)

-- ==========================================
-- 4. ABC MOVTOS (DETALLE)
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_abc_ingresos_devengo_movtos(
    p_accion INT,
    p_ingreso_movto_id INT DEFAULT NULL,
    p_ingreso_id INT DEFAULT NULL,
    p_centro_costo_id INT DEFAULT NULL,
    p_concepto_id INT DEFAULT NULL,
    p_monto_organizacion DECIMAL DEFAULT NULL,
    p_usuario VARCHAR DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_id INT;
BEGIN
    IF p_accion = 1 THEN
        IF p_ingreso_movto_id IS NULL OR p_ingreso_movto_id = 0 THEN
            INSERT INTO ingresos.ing_ingresos_movtos (ingreso_id, centro_costo_id, concepto_id, monto_organizacion, estatus_registro)
            VALUES (p_ingreso_id, p_centro_costo_id, p_concepto_id, p_monto_organizacion, 1)
            RETURNING ingreso_movto_id INTO v_id;
        ELSE
            UPDATE ingresos.ing_ingresos_movtos SET centro_costo_id = p_centro_costo_id, concepto_id = p_concepto_id, monto_organizacion = p_monto_organizacion
            WHERE ingreso_movto_id = p_ingreso_movto_id;
            v_id := p_ingreso_movto_id;
        END IF;
    ELSIF p_accion = 3 THEN
        UPDATE ingresos.ing_ingresos_movtos SET estatus_registro = 0 WHERE ingreso_movto_id = p_ingreso_movto_id;
        v_id := p_ingreso_movto_id;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'id', v_id);
END;
$$;

7 Acciones de la Pantalla
  • Nuevo Oficio: Abre el modal para capturar los datos recibidos de SEFIPLAN. Se guarda como Borrador (amarillo).
  • Editar: Modifica la captura. Solo disponible en Borrador.
  • Autorizar Devengo: Fija el monto, genera la Cuenta por Cobrar contable y cambia el estatus a "Devengado" (verde), pasándole la batuta a la caja.

9 Reglas de Negocio Críticas
  • Inmutabilidad: Al hacer clic en "Autorizar Devengo", el registro se bloquea. Cualquier error de captura en un oficio ya devengado requiere una cancelación formal (contra-póliza).
  • Visibilidad a Caja: Un oficio NO aparecerá en el módulo `04.2.3 Registro de Ingresos (Recaudado)` a menos que tenga estatus `2=Devengado`.
10 Flujos de Integración (Impacto Transversal)

Alimenta directamente al módulo 04.2.3 informando a la caja central cuánto dinero se espera recibir en los próximos días/horas.
