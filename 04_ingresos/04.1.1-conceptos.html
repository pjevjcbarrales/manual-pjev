04.1.1

Catálogo de Conceptos de Ingreso

Catálogo base operativo para tipificar las entradas de dinero al Poder Judicial y enlazarlas a la cuenta CRI.

Tipo: Catálogo Base
1 Descripción General

Este submódulo corresponde al Catálogo Base operativo para tipificar la naturaleza de las entradas de dinero al Poder Judicial. Aunque el CONAC cuenta con un Clasificador por Rubros de Ingresos (CRI), el Concepto de Ingreso funciona como una nomenclatura interna más amigable y específica para los operadores de la Oficina de Caja.

Definición SQL (PostgreSQL)

CREATE OR REPLACE FUNCTION ingresos.fn_leer_cat_conceptos(
    p_accion integer, p_usuario varchar, p_id integer, p_busqueda varchar DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    IF p_accion = 0 THEN
        IF p_id > 0 THEN
            SELECT row_to_json(t) INTO v_resultado FROM (
                SELECT a.*, c.cc_cuentacompleta, c.cc_descripcion 
                FROM ingresos.cat_conceptos a 
                LEFT JOIN contabilidad.catalogo_cuentas c ON a.cri_cuenta_id = c.cc_id 
                WHERE a.ci_id = p_id
            ) t;
        ELSE
            SELECT jsonb_build_object('registros', COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb)) INTO v_resultado
            FROM (
                SELECT a.*, c.cc_cuentacompleta FROM ingresos.cat_conceptos a
                LEFT JOIN contabilidad.catalogo_cuentas c ON a.cri_cuenta_id = c.cc_id
                WHERE a.estatus_registro = 1 
                AND (p_busqueda IS NULL OR a.ci_descripcion ILIKE '%' || p_busqueda || '%' OR a.ci_clave ILIKE '%' || p_busqueda || '%')
            ) t;
        END IF;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

Definición SQL (PostgreSQL)

CREATE OR REPLACE FUNCTION ingresos.fn_crud_cat_conceptos(
    p_accion integer, p_usuario varchar, p_id integer, p_clave varchar, p_descripcion varchar, 
    p_tipo_ingreso char, p_cri_cuenta_id integer, p_req_oficio integer, p_req_conciliacion integer
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_id integer := p_id;
BEGIN
    IF p_accion = 1 THEN
        INSERT INTO ingresos.cat_conceptos(ci_clave, ci_descripcion, ci_tipo_ingreso, cri_cuenta_id, ci_requiere_oficio, ci_requiere_conciliacion, estatus_registro, usuario_c)
        VALUES (p_clave, p_descripcion, p_tipo_ingreso, p_cri_cuenta_id, p_req_oficio, p_req_conciliacion, 1, p_usuario) RETURNING ci_id INTO v_id;
    ELSIF p_accion = 2 THEN
        UPDATE ingresos.cat_conceptos SET ci_clave = p_clave, ci_descripcion = p_descripcion, ci_tipo_ingreso = p_tipo_ingreso, cri_cuenta_id = p_cri_cuenta_id, ci_requiere_oficio = p_req_oficio, ci_requiere_conciliacion = p_req_conciliacion, usuario_m = p_usuario, fecha_m = now() WHERE ci_id = p_id;
    ELSIF p_accion = 3 THEN
        UPDATE ingresos.cat_conceptos SET estatus_registro = 0, usuario_m = p_usuario, fecha_m = now() WHERE ci_id = p_id;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'Guardado', 'data', jsonb_build_object('id', v_id));
EXCEPTION WHEN OTHERS THEN
    IF SQLSTATE = '23505' THEN RETURN jsonb_build_object('cod_resultado', -1, 'msg_resultado', 'La clave ya existe.'); END IF;
    RETURN jsonb_build_object('cod_resultado', -2, 'msg_resultado', SQLERRM);
END;
$$;
  • trg_valida_cri_concepto: BEFORE INSERT/UPDATE. Verifica en catalogo_cuentas que la cuenta asignada pertenezca al Rubro 4 (Ingresos). Si no, aborta.
  • trg_protege_integridad_conceptos: BEFORE UPDATE (cuando acción es baja). Impide la inhabilitación del concepto si se encuentra referenciado en registros transaccionales "Pendientes de Conciliar".
