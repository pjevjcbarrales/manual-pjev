Registro de Ingresos (Recaudado)

Módulo transaccional de caja para la formalización contable y vinculación bancaria de los oficios de SEFIPLAN que han caído en firme a las cuentas de la institución.

Modo A — existe en código Tipo: Análisis Funcional (DOC-08)
1 Descripción General

Este submódulo atiende exclusivamente la fase dos (Tiempo 2) del proceso. Aquí llega la tesorería (Caja Central) con su estado de cuenta en mano para certificar que el dinero prometido por SEFIPLAN ya fue depositado.

El sistema lee todos los oficios previamente "Devengados" y permite al cajero seleccionarlos, indicar la fecha en que ingresó el dinero y a qué cuenta bancaria específica (Banco/Cuenta Recaudadora) fue dirigido. Esta acción mata la cuenta por cobrar y refleja el flujo de efectivo real en el SIAF.

2 Marco Normativo
Norma Obligación que genera
Ley General de Contabilidad Gubernamental (LGCG) Obliga a registrar el Ingreso Recaudado exclusivamente cuando el recurso económico ingresa físicamente a las cuentas institucionales o caja.
Norma CONAC Exige la cancelación de la Cuenta por Cobrar (1.1.2.4) registrada en el devengo, y el incremento del Activo Disponible en Bancos (1.1.1.2) por el mismo monto exacto.
3 Roles que intervienen
Rol Acción permitida Restricción
Cajero / Tesorería Vincular la cuenta bancaria, fecha y folio de operación a un oficio pendiente. Autoriza el registro (Genera Póliza de Recaudo). No puede modificar el monto del oficio. Si el depósito no coincide con el oficio, el registro debe ser devuelto/cancelado.
4 Modelo de Datos (Maestro - Detalle)

El esquema maestro-detalle fue definido en la etapa de Devengo. En esta etapa (Recaudado), el cajero únicamente interactúa con el encabezado para registrar el depósito bancario.

Definición SQL (PostgreSQL)

-- ==========================================
-- 1. LEER ENCABEZADO RECAUDADO
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_leer_ingresos_recaudado_encabezado(
    p_ejercicio INT,
    p_ingreso_id INT DEFAULT NULL,
    p_fecha_inicio DATE DEFAULT NULL,
    p_fecha_fin DATE DEFAULT NULL,
    p_estatus_documento INT DEFAULT NULL,
    p_oficio_sefiplan VARCHAR DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    SELECT COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb) INTO v_resultado
    FROM (
        SELECT m.ingreso_id, m.folio_documento, m.monto_total, b.descripcion AS cuenta_recaudadora_desc, m.estatus_documento
        FROM ingresos.ing_ingresos m
        LEFT JOIN ingresos.ing_cuentas_recaudadoras b ON m.cuenta_recaudadora_id = b.cuenta_recaudadora_id
        WHERE m.ejercicio = p_ejercicio AND m.estatus_registro = 1 AND m.estatus_documento IN (2, 4)
          AND (p_ingreso_id IS NULL OR m.ingreso_id = p_ingreso_id)
          AND (p_fecha_inicio IS NULL OR m.fecha_documento >= p_fecha_inicio)
          AND (p_fecha_fin IS NULL OR m.fecha_documento <= p_fecha_fin)
          AND (p_estatus_documento IS NULL OR m.estatus_documento = p_estatus_documento)
          AND (p_oficio_sefiplan IS NULL OR m.folio_documento ILIKE '%' || p_oficio_sefiplan || '%')
    ) t;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

Definición SQL (PostgreSQL)

-- ==========================================
-- 2. LEER MOVTOS RECAUDADO
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_leer_ingresos_recaudado_movtos(
    p_ingreso_id INT
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    SELECT COALESCE(jsonb_agg(row_to_json(t)), '[]'::jsonb) INTO v_resultado
    FROM (
        SELECT d.ingreso_movto_id, d.centro_costo_id, d.concepto_id, d.monto_organizacion,
               c.descripcion AS concepto_desc, o.descripcion AS centro_costo_desc
        FROM ingresos.ing_ingresos_movtos d
        LEFT JOIN ingresos.ing_conceptos c ON d.concepto_id = c.concepto_id
        LEFT JOIN configuracion.organizaciones o ON d.centro_costo_id = o.organizacion_id
        WHERE d.ingreso_id = p_ingreso_id AND d.estatus_registro = 1
    ) t;
    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'OK', 'data', v_resultado);
END;
$$;

@p_ingreso_idINTObligatorio.
@p_cuenta_recaudadora_idINTBanco receptor.
@p_fecha_depositoDATEFecha bancaria.
@p_folio_operacionVARCHARReferencia.

{cod_resultado}INT1 = Éxito, 0 = Error.

Definición SQL (PostgreSQL)

-- ==========================================
-- 3. ABC RECAUDADO
-- ==========================================
CREATE OR REPLACE FUNCTION ingresos.fn_abc_ingresos_recaudado_encabezado(
    p_ingreso_id INT,
    p_cuenta_recaudadora_id INT,
    p_fecha_deposito DATE,
    p_folio_operacion VARCHAR,
    p_usuario VARCHAR DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_poliza_id INT;
BEGIN
    UPDATE ingresos.ing_ingresos SET
        cuenta_recaudadora_id = p_cuenta_recaudadora_id,
        fecha_deposito = p_fecha_deposito,
        folio_operacion = p_folio_operacion,
        estatus_documento = 4, -- Recaudado
        usuario_m = p_usuario,
        fecha_m = CURRENT_TIMESTAMP
    WHERE ingreso_id = p_ingreso_id AND estatus_documento = 2;

    RETURN jsonb_build_object('cod_resultado', 1, 'msg_resultado', 'Cobro Registrado');
END;
$$;

9 Reglas de Negocio Críticas
  • Lectura Estricta: El cajero no puede alterar el importe del oficio originario (pantalla devengo). Si el estado de cuenta refleja que SEFIPLAN depositó un centavo menos, el oficio completo debe ser cancelado o recibir una nota contable compensatoria.
  • Filtro en Combobox: El selector de Cuenta Bancaria debe listar únicamente registros activos del módulo 04.1.2 Cuentas Recaudadoras (Cuentas ligadas a 1.1.1.2).
10 Flujos de Integración (Impacto Transversal)

Es la acción que inyecta saldo 100% líquido a los tableros de Disponibilidad Financiera del PJEV para su posterior dispersión o pago a proveedores.

11 Afectación Contable (Póliza de Recaudado)

Al presionar "Autorizar Depósito", se liquida el derecho de cobro anterior y se genera la siguiente póliza automática:
