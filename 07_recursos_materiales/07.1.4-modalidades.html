07.1.4

Catálogo de Modalidades

Catálogo base que define los tipos de procedimiento de compra (Ej. Adjudicación Directa, Licitación Pública, etc.).

Modo A — Catálogo Simple
1 Descripción General

El Catálogo de Modalidades permite parametrizar las formas legales mediante las cuales la Dirección de Recursos Materiales puede llevar a cabo una contratación o compra. Cada modalidad tiene implicaciones distintas en cuanto a los montos y los documentos o checklists exigidos normativamente.

2 Marco Normativo
  • Ley 539 de Adquisiciones, Arrendamientos y Servicios del Estado de Veracruz: Establece los procedimientos de adjudicación directa, invitación a cuando menos tres personas y licitación pública.
  • Presupuesto de Egresos: Establece los montos tope de actuación para cada modalidad de acuerdo al ejercicio fiscal correspondiente.
3 Roles que intervienen
Rol Acción permitida Restricción
Administrador de Catálogos (RM) Alta, baja, cambio y consulta. No puede eliminar (baja física) una modalidad si ya existen requerimientos u oficios vinculados a ella. Solo procede la baja lógica.
Analista RM Solo consulta. Solo puede visualizar para uso dentro de los formularios operativos.

Definición SQL

-- Función para lectura
CREATE OR REPLACE FUNCTION materiales.fn_leer_modalidades(
    p_id_modalidad integer DEFAULT NULL,
    p_busqueda varchar DEFAULT NULL,
    p_estatus integer DEFAULT 1
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    IF p_id_modalidad > 0 THEN
        SELECT row_to_json(t) INTO v_resultado 
        FROM materiales.cat_modalidades t 
        WHERE t.id_modalidad = p_id_modalidad;
    ELSE
        SELECT jsonb_build_object('registros', COALESCE(jsonb_agg(row_to_json(l)), '[]'::jsonb)) INTO v_resultado
        FROM materiales.cat_modalidades l
        WHERE (p_busqueda IS NULL OR l.descripcion ILIKE '%' || p_busqueda || '%')
          AND (p_estatus IS NULL OR l.estatus_registro = p_estatus);
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'OK', 'data', v_resultado);
EXCEPTION WHEN OTHERS THEN
    RETURN jsonb_build_object('cod_resultado', -1, 'msg_resultado', SQLERRM);
END;
$$;
Parámetros de Entrada (IN)TipoDescripción
@p_accionINT1: Alta, 2: Modificación, 3: Baja Lógica
@p_id_modalidadINTRequerido para accion 2 y 3.
@p_descripcionVARCHARNombre de la modalidad.

Definición SQL

-- Función para guardado y edición
CREATE OR REPLACE FUNCTION materiales.fn_abc_modalidades(
    p_accion integer,
    p_id_modalidad integer DEFAULT NULL,
    p_descripcion varchar DEFAULT NULL
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_id integer;
BEGIN
    IF p_accion = 1 THEN
        INSERT INTO materiales.cat_modalidades (descripcion, estatus_registro)
        VALUES (p_descripcion, 1) RETURNING id_modalidad INTO v_id;
    ELSIF p_accion = 2 THEN
        UPDATE materiales.cat_modalidades SET descripcion = p_descripcion
        WHERE id_modalidad = p_id_modalidad;
        v_id := p_id_modalidad;
    ELSIF p_accion = 3 THEN
        UPDATE materiales.cat_modalidades SET estatus_registro = 0
        WHERE id_modalidad = p_id_modalidad;
        v_id := p_id_modalidad;
    END IF;
    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'Guardado correctamente', 'data', jsonb_build_object('id_modalidad', v_id));
EXCEPTION WHEN OTHERS THEN
    RETURN jsonb_build_object('cod_resultado', -1, 'msg_resultado', SQLERRM);
END;
$$;

7 Acciones del Módulo
  • Creación de Modalidades: Permite registrar nuevas formas de compra avaladas por la Ley.
  • Edición: Permite corregir la descripción textual.
  • Baja Lógica: Oculta la modalidad de los dropdowns operativos para no ser utilizada en nuevos requerimientos, manteniendo el historial.

9 Reglas de Negocio Especiales
  • Unicidad: No pueden existir dos modalidades activas con la misma descripción textual (Validación `UNIQUE` en DB o en código).
  • Integridad: Antes de proceder con la baja lógica, el Front-End puede preguntar "¿Estás seguro?", pero la Base de Datos es quien debe asegurar que no haya transacciones huérfanas mediante llaves foráneas.
10 Validaciones de Interfaz
  • El campo descripcion es obligatorio y no puede estar vacío ni contener puros espacios en blanco.
  • Límite de longitud a 255 caracteres en el input del modal.
11 Reportes y Salidas Físicas

No aplica para este catálogo simple. Se exporta a Excel como grid convencional en caso de que el componente base lo soporte.

12 Notificaciones y Alertas
