07.1.6

Configuración de Matriz (Checklists)

Motor de configuración dinámica que asocia una Modalidad de Compra con los N Documentos Requisito exigibles normativamente.

Tipo: Formulario Puente (Asignación Múltiple)
1 Descripción General

A diferencia de los catálogos simples, esta "Matriz" sirve para enlazar registros. Cuando el usuario operativo crea un nuevo Requerimiento y le asigna la modalidad "Licitación Pública", el sistema leerá de esta matriz exactamente cuáles documentos PDF le va a exigir adjuntar en el expediente. Es el puente entre el catálogo 07.1.4 y el 07.1.5.

2 Marco Normativo
  • Políticas y Bases de Adquisiciones: Define explícitamente qué papeles integran el expediente único para cada procedimiento.

Definición SQL

-- Función para leer la configuración
CREATE OR REPLACE FUNCTION materiales.fn_leer_matriz(
    p_id_modalidad integer
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE v_resultado jsonb;
BEGIN
    SELECT jsonb_build_object('asignados', COALESCE(jsonb_agg(row_to_json(m)), '[]'::jsonb)) INTO v_resultado
    FROM materiales.cat_matriz_checklist m
    JOIN materiales.cat_documentos_requisito d ON m.id_documento = d.id_documento
    WHERE m.id_modalidad = p_id_modalidad
      AND d.estatus_registro = 1;

    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'OK', 'data', v_resultado);
EXCEPTION WHEN OTHERS THEN
    RETURN jsonb_build_object('cod_resultado', -1, 'msg_resultado', SQLERRM);
END;
$$;

Definición SQL

-- Función para guardado masivo (Reemplazo total por modalidad)
CREATE OR REPLACE FUNCTION materiales.fn_guardar_matriz(
    p_id_modalidad integer,
    p_json_documentos jsonb
) RETURNS jsonb LANGUAGE plpgsql AS $$
DECLARE 
    v_doc_id integer;
BEGIN
    -- 1. Borrar asignaciones anteriores para esta modalidad
    DELETE FROM materiales.cat_matriz_checklist WHERE id_modalidad = p_id_modalidad;
    
    -- 2. Insertar las nuevas basado en el arreglo JSON de checkboxes
    FOR v_doc_id IN SELECT value::integer FROM jsonb_array_elements_text(p_json_documentos)
    LOOP
        INSERT INTO materiales.cat_matriz_checklist (id_modalidad, id_documento)
        VALUES (p_id_modalidad, v_doc_id);
    END LOOP;

    RETURN jsonb_build_object('cod_resultado', 0, 'msg_resultado', 'Matriz actualizada correctamente');
EXCEPTION WHEN OTHERS THEN
    RETURN jsonb_build_object('cod_resultado', -1, 'msg_resultado', SQLERRM);
END;
$$;

9 Reglas de Negocio Especiales

El guardado masivo (Reemplazo total) es la estrategia recomendada por simplicidad en el FrontEnd. Se eliminan todas las configuraciones para la modalidad seleccionada y se reinsertan los IDs checkeados actualmente en pantalla.

10 Validaciones de Interfaz
  • Validar que exista al menos 1 documento seleccionado antes de guardar.
  • Si el usuario cambia de Modalidad en el Combobox y tiene cambios sin guardar en los checkboxes, advertirle antes de refrescar los datos.
